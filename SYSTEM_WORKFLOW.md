# 🔄 ระบบการทำงานทั้งหมด - Slip Verification System

## 📊 ภาพรวมระบบ

ระบบนี้เป็นระบบตรวจสอบสลิปธนาคารอัตโนมัติที่เชื่อมต่อกับ LINE OA และ Google Sheets

```
┌─────────────────────────────────────────────────────────────────────┐
│                         LINE OA (ผู้ใช้)                            │
│                                                                     │
│  ผู้ใช้ส่งรูปภาพสลิป → LINE OA → Webhook → Server                  │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│                    Slip Verification Server                         │
│                                                                     │
│  1. รับ Webhook จาก LINE                                           │
│  2. ดาวน์โหลดรูปภาพ                                                │
│  3. สแกน QR Code                                                   │
│  4. ตรวจสอบสลิป (Slip2Go API)                                     │
│  5. บันทึกลง Google Sheets                                         │
│  6. ส่งข้อความตอบกลับ                                              │
└─────────────────────────────────────────────────────────────────────┘
                    ↙              ↓              ↘
        ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
        │ Slip2Go API  │  │Google Sheets │  │  LINE API    │
        │ (ตรวจสอบ)   │  │ (บันทึก)     │  │ (ส่งข้อความ) │
        └──────────────┘  └──────────────┘  └──────────────┘
```

## 🎮 ขั้นตอนการทำงานโดยละเอียด

### ขั้นที่ 1️⃣: ผู้ใช้ส่งรูปภาพสลิป

```
ผู้ใช้ → LINE OA
  ↓
  ส่งรูปภาพสลิป
  ↓
  LINE OA บันทึก Event
```

**สิ่งที่เกิดขึ้น:**
- ผู้ใช้เปิด LINE OA
- ส่งรูปภาพสลิปธนาคาร
- LINE OA รับรูปภาพ

---

### ขั้นที่ 2️⃣: LINE OA ส่ง Webhook ไปยัง Server

```
LINE OA → POST /webhook/line-slip-verification
  ↓
  ส่ง Event ที่มี:
  - userId (ผู้ใช้)
  - messageId (ID ของรูปภาพ)
  - type: "image"
```

**Request Body:**
```json
{
  "events": [
    {
      "type": "message",
      "source": {
        "userId": "U1234567890abcdef1234567890abcdef",
        "groupId": "C1234567890abcdef1234567890abcdef"
      },
      "message": {
        "type": "image",
        "id": "100001"
      }
    }
  ]
}
```

---

### ขั้นที่ 3️⃣: Server รับ Webhook

```
lineSlipVerificationWebhook.js
  ↓
  1. ตรวจสอบ Event Type
  2. ตรวจสอบว่าเป็น Image Message
  3. ดึง messageId
```

**Code:**
```javascript
if (event.type !== 'message') return;
if (event.message.type !== 'image') return;

const messageId = event.message.id;
const userId = event.source.userId;
```

---

### ขั้นที่ 4️⃣: ดาวน์โหลดรูปภาพจาก LINE

```
Server → LINE API
  ↓
  GET /v2/bot/message/{messageId}/content
  ↓
  ได้ Image Buffer
```

**สิ่งที่เกิดขึ้น:**
- ใช้ messageId เพื่อดึงรูปภาพ
- ได้ Buffer ของรูปภาพ
- เก็บไว้ในหน่วยความจำ

---

### ขั้นที่ 5️⃣: สแกน QR Code จากรูปภาพ

```
QRCodeScannerService
  ↓
  1. อ่านรูปภาพ (ใช้ Jimp)
  2. แปลงเป็น ImageData
  3. สแกน QR Code (ใช้ jsQR)
  ↓
  ได้ QR Code String
  เช่น: "0041000600000101030040220014242082547BPM049885102TH9104xxxx"
```

**ตัวอย่าง:**
```javascript
const scanner = new QRCodeScannerService();
const qrCode = await scanner.scanQRCodeFromBuffer(imageBuffer);
// qrCode = "0041000600000101030040220014242082547BPM049885102TH9104xxxx"
```

---

### ขั้นที่ 6️⃣: ตรวจสอบสลิปผ่าน Slip2Go API

```
Slip2GoQRVerificationService
  ↓
  POST https://api.slip2go.com/api/verify-slip/qr-code/info
  ↓
  Headers:
    Authorization: Bearer <SECRET_KEY>
    Content-Type: application/json
  ↓
  Body:
  {
    "payload": {
      "qrCode": "0041000600000101030040220014242082547BPM049885102TH9104xxxx",
      "checkCondition": {
        "checkDuplicate": true,
        "checkReceiver": [
          {
            "accountNumber": "xxxxxx1234",
            "accountType": "01004"
          }
        ]
      }
    }
  }
  ↓
  Response:
  {
    "code": "200000",
    "message": "Slip found",
    "data": {
      "referenceId": "92887bd5-60d3-4744-9a98-b8574eaxxxxx",
      "amount": 1000,
      "dateTime": "2025-10-05T14:48:00.000Z",
      "transRef": "015073144041ATF00999",
      "sender": { ... },
      "receiver": { ... }
    }
  }
```

**ผลลัพธ์:**
- ✅ code = "200000" → สลิปถูกต้อง
- ❌ code = "200501" → สลิปซ้ำ
- ❌ code = "200401" → บัญชีผู้รับไม่ถูกต้อง

### ขั้นที่ 7️⃣: ตรวจสอบสลิปซ้ำ (Optional)

```
  ↓

  ↓
  ถ้าพบ → สลิปซ้ำ ❌
  ถ้าไม่พบ → สลิปใหม่ ✅
```

### ขั้นที่ 8️⃣: บันทึกข้อมูลลง Google Sheets

```
SlipRecordingService
  ↓
  1. ตรวจสอบ Worksheet มีอยู่หรือไม่
  2. ถ้าไม่มี → สร้าง Worksheet ใหม่
  3. เพิ่ม Header (ถ้าเป็นครั้งแรก)
  4. บันทึกข้อมูลแถวใหม่
  ↓
  Google Sheets API
  ↓
  Append values to sheet
```

**ข้อมูลที่บันทึก:**
```
วันที่บันทึก | Reference ID | Transaction Ref | จำนวนเงิน | วันที่โอน | ชื่อผู้ส่ง | ... | สถานะ
26/02/2025   | 92887bd5-... | 015073144041... | 1000      | 2025-10-05| สมชาย    | ... | verified
```

## 🎮 ระบบการเล่นทั้งหมด

### 1. ผู้ใช้ (User)
- ส่งรูปภาพสลิปไปยัง LINE OA
- รับข้อความตอบกลับ

### 2. LINE OA
- รับรูปภาพจากผู้ใช้
- ส่ง Webhook ไปยัง Server
- ส่งข้อความตอบกลับไปยังผู้ใช้

### 3. Server (Node.js)
- รับ Webhook จาก LINE
- ดาวน์โหลดรูปภาพ
- สแกน QR Code
- ตรวจสอบสลิป
- บันทึกข้อมูล
- ส่งข้อความตอบกลับ

### 4. Slip2Go API
- ตรวจสอบความถูกต้องของสลิป
- ส่งข้อมูลสลิปกลับมา

### 5. Google Sheets
- บันทึกข้อมูลสลิป
- ให้ค้นหาข้อมูลได้

## 🔐 ความปลอดภัย

### 1. Authentication
- LINE: ตรวจสอบ Channel Secret
- Slip2Go: ใช้ Bearer Token
- Google Sheets: ใช้ Service Account

### 2. Data Protection
- Secret Key เก็บใน `.env`
- ไม่ commit credentials ไปยัง Git
- ใช้ HTTPS สำหรับ Webhook URL

### 3. Validation
- ตรวจสอบ Event Type
- ตรวจสอบ Message Type
- ตรวจสอบ QR Code Format

## 🎯 สรุป

ระบบนี้ทำงานแบบ **Real-time** โดยมีขั้นตอนดังนี้:

1. **รับ** - รับรูปภาพจาก LINE OA
2. **ประมวลผล** - สแกน QR Code และตรวจสอบสลิป
3. **บันทึก** - บันทึกข้อมูลลง Google Sheets
4. **ตอบกลับ** - ส่งข้อความตอบกลับไปยัง LINE OA

ทั้งหมดนี้เกิดขึ้นในเวลาประมาณ **10-15 วินาที** ✅
