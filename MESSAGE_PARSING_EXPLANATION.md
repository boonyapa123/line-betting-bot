# การถอดและบันทึกข้อความ ไล่/360-400/20เป็ด

## รูปแบบข้อความ (Message Format)
```
ไล่/360-400/20เป็ด
```

### ส่วนประกอบ:
- **ไล่** = ฝั่ง (Side Code) - ไล่ (ล) หรือ ยั้ง (ต)
- **360-400** = ช่วงราคา (Price Range)
- **20** = ยอดเงิน (Amount)
- **เป็ด** = ชื่อบั้งไฟ (Slip Name)

---

## STEP 1: User A ส่งข้อความ

### ข้อความ:
```
ไล่/360-400/20เป็ด
```

### ระบบถอด (Parse):
```javascript
{
  success: true,
  method: 2,              // Method 2 = Slash Format
  slipName: "เป็ด",       // ✅ ชื่อบั้งไฟ
  price: "360-400",       // ✅ ช่วงราคา
  sideCode: "ล",          // ✅ ฝั่ง (ไล่)
  amount: 20              // ✅ ยอดเงิน
}
```

### บันทึกลงชีท Bets:
| Column | ชื่อ | ค่า |
|--------|------|-----|
| D | MESSAGE_A | ไล่/360-400/20เป็ด |
| E | SLIP_NAME | **เป็ด** ✅ |
| F | SIDE_A | 360-400 ล |
| G | AMOUNT | 20 |

---

## STEP 2: User B Reply ข้อความเดียวกัน

### ข้อความ:
```
ไล่/360-400/20เป็ด
```

### ระบบทำงาน:
1. ตรวจสอบว่า User B reply ข้อความของ User A ✅
2. ค้นหา pending bet ของ User A ✅
3. ดึงข้อมูล User A จากชีท:
   - Slip Name: **เป็ด** ✅
   - Price: 360-400
   - Side: ล
   - Amount: 20

4. คำนวณฝั่งตรงข้าม:
   - User A: ล (ไล่)
   - User B: ต (ยั้ง) ✅

5. สร้าง Price B:
   - Price: 360-400
   - Side: ต (ยั้ง)
   - Result: **360-400 ต** ✅

6. อัปเดตแถว User A ด้วยข้อมูล User B:
   - **ใช้ Slip Name เดียวกับของ User A: เป็ด** ✅

### บันทึกลงชีท Bets (หลังจับคู่):
| Column | ชื่อ | User A | User B |
|--------|------|--------|--------|
| D | MESSAGE_A | ไล่/360-400/20เป็ด | - |
| E | SLIP_NAME | **เป็ด** ✅ | - |
| F | SIDE_A | 360-400 ล | - |
| G | AMOUNT | 20 | - |
| H | AMOUNT_B | - | 20 |
| L | USER_B_NAME | - | 💓Noon💓 |
| M | SIDE_B | - | 360-400 ต |
| U | MATCHED_AUTO | - | AUTO |

---

## สรุป

### ✅ Slip Name ถูกต้อง:
- **User A ส่ง**: ไล่/360-400/20**เป็ด**
- **ระบบถอด**: Slip Name = **เป็ด** ✅
- **บันทึกลงชีท**: Column E = **เป็ด** ✅
- **User B reply**: Slip Name ยังคงเป็น **เป็ด** ✅

### ✅ Price ถูกต้อง:
- **User A**: 360-400 ล
- **User B**: 360-400 ต (ฝั่งตรงข้าม)

### ✅ Amount ถูกต้อง:
- **User A**: 20 บาท
- **User B**: 20 บาท (เท่ากับ User A)

---

## ข้อสำคัญ

1. **Slip Name ต้องเป็นเฉพาะชื่อบั้งไฟ** (ไม่มีราคา)
   - ✅ ถูก: `เป็ด`
   - ❌ ผิด: `360-400 เป็ด`

2. **เมื่อ User B reply ต้องใช้ Slip Name เดียวกับ User A**
   - ✅ ถูก: ใช้ `pendingBet.slipName`
   - ❌ ผิด: ใช้ `parsedBet.slipName` (อาจจะมาจากการ parse ข้อความ reply)

3. **Price B ต้องมีฝั่งตรงข้าม**
   - User A: `360-400 ล` → User B: `360-400 ต`
   - User A: `360-400 ต` → User B: `360-400 ล`

4. **Amount ต้องเท่ากัน**
   - User A: 20 บาท → User B: 20 บาท
