# ระบบบันทึกข้อมูลการจับคู่อัตโนมัติ

## 📋 ภาพรวม

เมื่อระบบจับคู่เล่นได้สำเร็จ จะบันทึกข้อมูลลงชีท Bets โดยอัตโนมัติ พร้อมกับส่งแจ้งเตือนไปยัง LINE

## 🔧 โครงสร้างโค้ด

### ไฟล์ที่เกี่ยวข้อง

1. **bettingRoundController.js**
   - ตรวจสอบการจับคู่อัตโนมัติ
   - เรียกฟังก์ชันบันทึกลงชีท

2. **priceRangeMatchingService.js**
   - `recordToGoogleSheets()` - บันทึกข้อมูลลงชีท Bets (20 คอลัมน์)

## 📊 ขั้นตอนการบันทึก

```
User B: 350-380 ย 300 ฟ้าหลังฝน
    ↓
[Parse Message] → method: 2, side: ย, price: 350-380, amount: 300
    ↓
[Record Bet] → บันทึกลงฐานข้อมูล
    ↓
[Check Balance] → ตรวจสอบยอดเงิน ✅
    ↓
[Find Match] → ค้นหาคู่ที่มีฝั่งตรงข้าม
    ↓
    ✅ พบ User A → จับคู่อัตโนมัติ
    ↓
[Record to Sheets]
    ├─ ดึงจำนวนแถวปัจจุบัน
    ├─ สร้างข้อมูลแถว (20 คอลัมน์)
    └─ บันทึกลงชีท Bets (A:T)
    ↓
[Create Messages] → สร้างข้อความแจ้งเตือน
    ├─ messageA: ข้อความสำหรับ User A
    ├─ messageB: ข้อความสำหรับ User B
    └─ groupMessage: ข้อความสำหรับกลุ่ม
    ↓
[Send Notifications]
    ├─ 📤 ส่งข้อความส่วนตัวให้ User A
    ├─ 📤 ส่งข้อความส่วนตัวให้ User B
    └─ 📢 ส่งข้อความเข้ากลุ่ม (ถ้ามี groupId)
    ↓
✅ บันทึกและแจ้งเตือนสำเร็จ
```

6. ✅ ตรงกับโครงสร้างชีท Bets ปัจจุบัน

## 📝 หมายเหตุ

- ข้อมูลจะบันทึกลงชีท Bets ทันทีเมื่อจับคู่ได้
- ยอดเงินที่บันทึกจะใช้ยอดน้อยกว่าของทั้งสองฝั่ง
- ผลแพ้ชนะจะอัปเดตเมื่อประกาศผลเท่านั้น
- Timestamp จะใช้เวลาปัจจุบันของระบบ (เวลาไทย)
- บันทึกลงคอลัมน์ A ถึง T (20 คอลัมน์)
