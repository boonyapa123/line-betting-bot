# 🔐 ระบบตรวจสอบสลิป (ฉบับสุดท้าย)

## 📋 ภาพรวม

ระบบตรวจสอบสลิปจะตรวจสอบ **3 เงื่อนไข** ผ่าน Slip2Go API:

1. **สลิปซ้ำ (Duplicate Check)** - ตรวจสอบว่าสลิปนี้เคยบันทึกไปแล้วหรือไม่
2. **บัญชีตรงกันหรือไม่ (Receiver Account Check)** - ตรวจสอบว่าบัญชีผู้รับตรงกับบัญชีที่ตั้งค่าไว้
3. **ชื่อผู้รับ (Receiver Name Check)** - ตรวจสอบว่ามีชื่อผู้รับในสลิป

**ยอดเงินโอนเท่าไหร่ก็ได้** ✅ ไม่มีการตรวจสอบจำนวนเงิน

---

## 🔧 Response Code จาก Slip2Go API

| Code | ความหมาย | การจัดการ |
|------|---------|---------|
| 200000 | ✅ สลิปถูกต้อง (ไม่ได้ตรวจสอบเงื่อนไข) | บันทึกข้อมูล |
| 200200 | ✅ สลิปถูกต้อง + บัญชีตรงกัน | บันทึกข้อมูล |
| 200100 | ❌ สลิปซ้ำ | ปฏิเสธและแจ้งเหตุผล |
| 200300 | ❌ บัญชีไม่ตรงกัน | ปฏิเสธและแจ้งเหตุผล |

---

## 🔄 การไหลของข้อมูล

```
ผู้ใช้ส่งสลิป
    ↓
ดาวน์โหลดรูปภาพจาก LINE
    ↓
ส่งไปยัง Slip2Go API พร้อม checkCondition:
  - checkDuplicate: true
  - checkReceiver: [{ accountType, accountNumber }]
    ↓
Slip2Go API ตรวจสอบ:
  1. สลิปซ้ำหรือไม่ (Code: 200100)
  2. บัญชีตรงกันหรือไม่ (Code: 200300)
    ↓
ได้รับ Response Code:
  - 200000 = ✅ สลิปถูกต้อง
  - 200200 = ✅ สลิปถูกต้อง + บัญชีตรงกัน
  - 200100 = ❌ สลิปซ้ำ
  - 200300 = ❌ บัญชีไม่ตรงกัน
    ↓
ถ้า 200000 หรือ 200200 → บันทึกข้อมูล
ถ้า 200100 หรือ 200300 → ปฏิเสธและแจ้งเหตุผล
```

## 🔧 ตัวแปร Environment

เพิ่มตัวแปรต่อไปนี้ใน `.env`:

```env
# Slip2Go Configuration
SLIP2GO_SECRET_KEY=your_secret_key
SLIP2GO_API_URL=https://api.slip2go.com

# Slip Verification (ตั้งค่าเสมอ)
SLIP_CHECK_DUPLICATE=true          # ตรวจสอบสลิปซ้ำ
SLIP_CHECK_RECEIVER=true           # ตรวจสอบบัญชีผู้รับ
```

## 🚀 การใช้งาน

ระบบตรวจสอบสลิปใหม่จะทำงานโดยอัตโนมัติเมื่อผู้ใช้ส่งรูปภาพสลิป:

1. ระบบจะดาวน์โหลดรูปภาพจาก LINE
2. ส่งไปยัง Slip2Go API พร้อม checkCondition
3. Slip2Go API ตรวจสอบ:
   - สลิปซ้ำหรือไม่
   - บัญชีตรงกันหรือไม่
4. ถ้าตรวจสอบสำเร็จ (Code: 200000 หรือ 200200) → บันทึกข้อมูล
5. ถ้าตรวจสอบไม่สำเร็จ → ปฏิเสธและแจ้งเหตุผล

## 🐛 Troubleshooting

### ปัญหา: สลิปถูกปฏิเสธแม้ว่าถูกต้อง

**วิธีแก้:**
1. ตรวจสอบ Response Code จาก Slip2Go API ในบันทึก
2. ตรวจสอบว่าบัญชีผู้รับตรงกัน
3. ตรวจสอบว่าสลิปไม่ซ้ำ

### ปัญหา: สลิปซ้ำถูกตรวจสอบไม่ถูกต้อง

**วิธีแก้:**
1. ตรวจสอบว่า `checkDuplicate: true` ถูกตั้งค่า
2. ตรวจสอบ Response Code (ควรเป็น 200100)

### ปัญหา: บัญชีไม่ตรงกัน

**วิธีแก้:**
1. ตรวจสอบว่าบัญชีผู้รับถูกต้อง
2. ตรวจสอบ Response Code (ควรเป็น 200300)
