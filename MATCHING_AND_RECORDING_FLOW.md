# ขั้นตอนการจับคู่และบันทึกข้อมูลลงชีท Bets

## 1️⃣ ขั้นตอนการจับคู่ (Matching)

### ที่มา: `bettingMatchingService.js` - `findPairsAndDeductBalance()`

**ขั้นตอน:**
1. ดึงข้อมูลการเล่นทั้งหมดจากชีท Bets
2. ค้นหาคู่ที่ตรงกัน (ตรวจสอบ: บั้งไฟ, ฝั่ง, ยอดเงิน)
3. หักเงินจากทั้งสองฝั่ง
4. บันทึกข้อมูลการจับคู่ลงชีท

---

## 2️⃣ บันทึกข้อมูลการจับคู่ลงชีท Bets

### ที่มา: `index.js` - ฟังก์ชันที่จัดการการจับคู่

**ข้อมูลที่บันทึก (18 คอลัมน์):**

| คอลัมน์ | ชื่อ | ข้อมูลที่บันทึก | หมายเหตุ |
|--------|------|----------------|---------|
| A | วันเวลา | `new Date().toISOString()` | วันเวลาที่จับคู่ |
| B | ID ผู้เล่น A | `pair.userA` | ID ของผู้เล่นคนแรก |
| C | ชื่อผู้เล่น A | `userAName` | ชื่อ LINE ของผู้เล่นคนแรก |
| D | ข้อความ | `pair.messageA` | ข้อความเดิมพันที่ส่ง |
| E | ชื่อบั้งไฟ | `betDetailsA.fireworkName` | ชื่อบั้งไฟ (เช่น "ลูกชายภูน้อย") |
| F | ฝั่ง A | `betDetailsA.betType` | ฝั่งที่เดิมพัน (ชล/ถอย/ล/ย) |
| G | เงิน A | `betAmount` | ยอดเงินที่เดิมพัน |
| H | เงิน B | `betAmount` | ยอดเงินของผู้เล่นคนที่สอง (เท่ากับ A) |
| I | ผลลัพธ์ | `''` (ว่างเปล่า) | รอการประกาศผล |
| J | ผู้ชนะ | `''` (ว่างเปล่า) | รอการประกาศผล |
| K | ID ผู้เล่น B | `pair.userB` | ID ของผู้เล่นคนที่สอง |
| L | ชื่อผู้เล่น B | `userBName` | ชื่อ LINE ของผู้เล่นคนที่สอง |
| M | ฝั่ง B | `oppositeBetType` | ฝั่งตรงข้าม (ชถ/ไล่/ย/ล) |
| N | ฝั่ง A (รหัส) | `betDetailsA.betType` | รหัสฝั่ง A |
| O | ชื่อกลุ่ม | `groupName` | ชื่อกลุ่ม LINE |
| P | Token A | `userAToken` | Access Token ของผู้เล่น A |
| Q | ID กลุ่ม | `pair.groupId` | ID ของกลุ่ม LINE |
| R | Token B | `userBToken` | Access Token ของผู้เล่น B |

## 3️⃣ ตัวอย่างข้อมูลที่บันทึก

### ก่อนจับคู่ (User A เดิมพัน):
```
A: 2024-03-04T10:30:00.000Z
B: U001
C: John
D: ลูกชายภูน้อย ชล 150
E: ลูกชายภูน้อย
F: ชล
G: 150
H: (ว่างเปล่า)
I: (ว่างเปล่า)
J: (ว่างเปล่า)
K: (ว่างเปล่า)
L: (ว่างเปล่า)
M: (ว่างเปล่า)
N: ชล
O: (ว่างเปล่า)
P: token_A
Q: (ว่างเปล่า)
R: (ว่างเปล่า)
```

### หลังจับคู่ (User B ตอบรับ):
```
A: 2024-03-04T10:30:00.000Z
B: U001
C: John
D: ลูกชายภูน้อย ชล 150
E: ลูกชายภูน้อย
F: ชล
G: 150
H: 150                    ← เพิ่มยอดเงิน B
I: (ว่างเปล่า)
J: (ว่างเปล่า)
K: U002                   ← เพิ่ม ID ผู้เล่น B
L: Jane                   ← เพิ่มชื่อผู้เล่น B
M: ชถ                     ← เพิ่มฝั่ง B (ตรงข้าม)
N: ชล
O: ทดสอบระบบ บั้งไฟ เดิมพัน
P: token_A
Q: C4e5222774807...      ← เพิ่ม ID กลุ่ม
R: token_B                ← เพิ่ม Token B
```

## 4️⃣ ขั้นตอนการบันทึกผลลัพธ์

### ที่มา: `bettingResultService.js` - `recordResult()`

**เมื่อประกาศผล:**
- **I (ผลลัพธ์)**: บันทึกคะแนนที่ออก (เช่น "ชล")
- **J (ผู้ชนะ)**: บันทึกชื่อผู้ชนะ
- **S (ผลลัพธ์ A)**: WIN/LOSE/DRAW
- **T (ผลลัพธ์ B)**: WIN/LOSE/DRAW

---

## 5️⃣ สรุปการไหลของข้อมูล

```
1. User A เดิมพัน
   ↓
   บันทึก: A-G, N, O, P, Q (ส่วนที่เหลือว่างเปล่า)

2. ระบบค้นหาคู่
   ↓
   User B ตอบรับ

3. ระบบจับคู่สำเร็จ
   ↓
   อัปเดต: H, K, L, M, R (ข้อมูล User B)

4. ประกาศผล
   ↓
   อัปเดต: I, J, S, T (ผลลัพธ์)

5. อัปเดตยอดเงิน
   ↓
   ชีท UsersBalance: อัปเดตยอดเงินของ A และ B
```

---

## 6️⃣ ฟังก์ชันที่เกี่ยวข้อง

| ฟังก์ชัน | ไฟล์ | หน้าที่ |
|---------|------|--------|
| `recordBet()` | `bettingPairingService.js` | บันทึกการเล่นครั้งแรก (User A) |
| `findPairsAndDeductBalance()` | `bettingMatchingService.js` | จับคู่และหักเงิน |
| `recordResult()` | `bettingResultService.js` | บันทึกผลลัพธ์ |
| `updateUserBalance()` | `balanceUpdateService.js` | อัปเดตยอดเงิน |
