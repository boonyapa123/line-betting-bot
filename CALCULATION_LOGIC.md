## 2️⃣ วิธีการเดิมพัน 3 แบบ

### 📌 วิธีที่ 1: REPLY Method (ตอบรับ)
- **ลักษณะ**: ผู้เล่นตอบรับโดยไม่ระบุฝั่ง
- **การคำนวน**: ให้ผู้เล่นคนแรก (bet1) ชนะ
- **ยอดเงิน**: 0 (ไม่มีการเดิมพัน)

**ตัวอย่าง:**
```
User A: "ลูกชายภูน้อย ชล 150"
User B: "ต" (ตอบรับ)
→ User A ชนะ
```

---

### 📌 วิธีที่ 2: Direct Method 1 (ไม่มีราคา)
- **ลักษณะ**: ทั้งสองฝั่งระบุฝั่งชัดเจน (ชล/ชถ/ล/ย)
- **การคำนวน**: ฝั่ง "ชล" (ไล่) ชนะ
- **ยอดเงิน**: ใช้ยอดน้อยกว่า

**ตัวอย่าง:**
```
User A: "ลูกชายภูน้อย ชล 150"
User B: "ลูกชายภูน้อย ชถ 100"
→ User A ชนะ (ฝั่ง ชล)
→ ยอดเงิน = min(150, 100) = 100 บาท
```

---

### 📌 วิธีที่ 3: Direct Method 2 (มีราคา)
- **ลักษณะ**: ระบุราคา (เกณฑ์คะแนน)
- **การคำนวน**: ตรวจสอบคะแนนกับเกณฑ์
  - **คะแนนอยู่ในเกณฑ์** → ฝั่ง "ล" (ไล่) ชนะ
  - **คะแนนไม่อยู่ในเกณฑ์** → ฝั่ง "ย" (ยั้ง) ชนะ
- **ยอดเงิน**: ใช้ยอดน้อยกว่า

**ตัวอย่าง:**
```
User A: "ลูกชายภูน้อย ล 0/3(300-330) 150"
User B: "ลูกชายภูน้อย ย 0/3(300-330) 100"

ถ้าคะแนนออก 315:
→ 315 อยู่ในเกณฑ์ (300-330)
→ User A ชนะ (ฝั่ง ล)
→ ยอดเงิน = min(150, 100) = 100 บาท

ถ้าคะแนนออก 250:
→ 250 ไม่อยู่ในเกณฑ์ (300-330)
→ User B ชนะ (ฝั่ง ย)
→ ยอดเงิน = min(150, 100) = 100 บาท
```

---

## 3️⃣ การคำนวนค่าธรรมเนียม

### ที่มา: `bettingResultService.js` - `calculateResultWithFees()`

#### 🏆 ชนะ-แพ้ (WIN-LOSE)
```
ยอดเงินเดิมพัน = 100 บาท

ค่าธรรมเนียม = 100 × 10% = 10 บาท

ผู้ชนะ:
  - Gross Amount = 100 บาท (ยอดเงินเดิมพัน)
  - Fee = 10 บาท (ค่าธรรมเนียม)
  - Net Amount = 100 - 10 = 90 บาท (ได้รับจริง)

ผู้แพ้:
  - Gross Amount = -100 บาท (เสีย)
  - Fee = 0 บาท
  - Net Amount = -100 บาท (เสียไป)
```

#### 🤝 ออกกลาง (DRAW)
```
ยอดเงินเดิมพัน = 100 บาท

ค่าธรรมเนียม = 100 × 5% = 5 บาท (ทั้งสองฝั่ง)

ผู้เล่น A:
  - Gross Amount = 0 บาท
  - Fee = 5 บาท (ค่าธรรมเนียม)
  - Net Amount = -5 บาท (เสีย)

ผู้เล่น B:
  - Gross Amount = 0 บาท
  - Fee = 5 บาท (ค่าธรรมเนียม)
  - Net Amount = -5 บาท (เสีย)
```

---

## 4️⃣ ตัวอย่างการคำนวนแบบเต็ม

### สถานการณ์: Direct Method 1 (ไม่มีราคา)

```
User A: "ลูกชายภูน้อย ชล 150"
User B: "ลูกชายภูน้อย ชถ 100"

Step 1: ตรวจสอบการจับคู่
  ✓ บั้งไฟเดียวกัน (ลูกชายภูน้อย)
  ✓ ฝั่งตรงข้าม (ชล vs ชถ)
  ✓ ยอดเงินเพียงพอ
  → จับคู่สำเร็จ

Step 2: คำนวนยอดเงิน
  ยอดเงิน = min(150, 100) = 100 บาท

Step 3: คำนวนผลลัพธ์
  ฝั่ง "ชล" ชนะ
  → User A ชนะ
  → User B แพ้

Step 4: คำนวนค่าธรรมเนียม
  ค่าธรรมเนียม = 100 × 10% = 10 บาท

  User A (ชนะ):
    - Gross: 100 บาท
    - Fee: 10 บาท
    - Net: 90 บาท

  User B (แพ้):
    - Gross: -100 บาท
    - Fee: 0 บาท
    - Net: -100 บาท

Step 5: อัปเดตยอดเงิน
  User A: 1000 + 90 = 1090 บาท
  User B: 800 - 100 = 700 บาท
```

## 5️⃣ ตรวจสอบการออกกลาง (DRAW)

### ที่มา: `bettingResultService.js` - `isDrawResult()`

**กรณีที่ถือว่าออกกลาง:**

1. **REPLY Method ทั้งสองฝั่ง**
   ```
   User A: "ลูกชายภูน้อย ชล 150"
   User B: "ต" (ตอบรับ)
   → ออกกลาง (DRAW)
   ```

2. **Direct Method 2 (มีราคา) - คะแนนอยู่ในเกณฑ์ของทั้งสองฝั่ง**
   ```
   User A: "ลูกชายภูน้อย ล 0/3(300-330) 150"
   User B: "ลูกชายภูน้อย ย 0/3(300-330) 100"
   
   คะแนนออก: 315
   → 315 อยู่ในเกณฑ์ของ A (300-330)
   → 315 อยู่ในเกณฑ์ของ B (300-330)
   → ออกกลาง (DRAW)
   ```

## 8️⃣ สรุปการคำนวน

```
1. ตรวจสอบการจับคู่ (บั้งไฟ, ฝั่ง, ยอดเงิน)
   ↓
2. คำนวนยอดเงิน (ใช้ยอดน้อยกว่า)
   ↓
3. คำนวนผลลัพธ์ (ตามวิธีการเดิมพัน)
   ↓
4. ตรวจสอบการออกกลาง
   ↓
5. คำนวนค่าธรรมเนียม (10% หรือ 5%)
   ↓
6. อัปเดตยอดเงิน
   ↓
7. บันทึกผลลัพธ์ลงชีท
   ↓
8. แจ้งเตือน LINE
```
