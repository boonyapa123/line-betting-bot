# คำสั่งแอดมิน - Admin Commands Guide

## 📋 ภาพรวม

ระบบมีคำสั่งแอดมิน 3 คำสั่งหลัก:
1. `:เริ่ม` - เปิดรอบการเล่น
2. `:หยุด` - ปิดรอบการเล่น
3. `:สรุป` - สรุปผลการเล่น

---

## 1️⃣ คำสั่ง :เริ่ม (Start Round)

### วัตถุประสงค์
เปิดรอบการเล่นใหม่ เพื่อให้ผู้เล่นสามารถส่งข้อความเล่นได้

### รูปแบบ
```
:เริ่ม <ชื่อบั้งไฟ>
```

### ตัวอย่าง
```
:เริ่ม บั้งไฟ1
:เริ่ม บั้งไฟ2
:เริ่ม บั้งไฟ3
```

### Response
```
✅ เปิดรอบการเล่น: บั้งไฟ1
รอบ ID: ROUND_1705756800000
```

### ขั้นตอนการทำงาน
1. ตรวจสอบรูปแบบคำสั่ง
2. ดึงชื่อบั้งไฟ
3. สร้าง Round ID
4. บันทึกลงชีท RoundState
5. เปลี่ยนสถานะเป็น OPEN
6. ส่งข้อความยืนยัน

### ข้อมูลที่บันทึก
- **State:** OPEN
- **RoundID:** ROUND_[timestamp]
- **StartTime:** [วันเวลาปัจจุบัน]
- **SlipName:** [ชื่อบั้งไฟ]

---

## 2️⃣ คำสั่ง :หยุด (Stop Round)

### วัตถุประสงค์
ปิดรอบการเล่น เพื่อไม่ให้ผู้เล่นส่งข้อความเล่นเพิ่มเติม

### รูปแบบ
```
:หยุด
```

### ตัวอย่าง
```
:หยุด
```

### Response
```
✅ รอบนี้ปิดการทายแล้วคะ/ครับ
```

### ขั้นตอนการทำงาน
1. ตรวจสอบรูปแบบคำสั่ง
2. เปลี่ยนสถานะเป็น CLOSED
3. บันทึกลงชีท RoundState
4. ส่งข้อความยืนยัน

### ข้อมูลที่บันทึก
- **State:** CLOSED

### ผลกระทบ
- ผู้เล่นไม่สามารถส่งข้อความเล่นได้
- ระบบจะตอบ "รอบนี้ปิดการทายแล้ว"

---

## 3️⃣ คำสั่ง :สรุป (Calculate Result)

### วัตถุประสงค์
สรุปผลการเล่น คำนวณผู้ชนะ ผู้แพ้ และอัปเดตยอดเงิน

### รูปแบบ
```
:สรุป <ชื่อบั้งไฟ> <คะแนน>
```

### ตัวอย่าง
```
:สรุป บั้งไฟ1 50
:สรุป บั้งไฟ2 45
:สรุป บั้งไฟ3 60
```

### Response
```
📊 สรุปผลการเล่น
บั้งไฟ: บั้งไฟ1
คะแนนที่ออก: 50
========================================

คู่ที่ 1:
🏆 ชนะ: สมชาย +450 บาท
❌ แพ้: สมหญิง -500 บาท

คู่ที่ 2:
🏆 ชนะ: สมศรี +450 บาท
❌ แพ้: สมปรารถนา -500 บาท

========================================
💰 ยอดเงินคงเหลือ:

สมชาย: 1450 บาท
สมหญิง: 500 บาท
สมศรี: 1450 บาท
สมปรารถนา: 500 บาท
```

### ขั้นตอนการทำงาน
1. ตรวจสอบรูปแบบคำสั่ง
2. เปลี่ยนสถานะเป็น CALCULATING
3. ดึงข้อมูลการเล่นทั้งหมด
4. กรองเฉพาะการเล่นของบั้งไฟนี้
5. จับคู่การเล่น
6. คำนวณผลลัพธ์และค่าธรรมเนียม
7. บันทึกลงชีท Results
8. อัปเดตยอดเงิน
9. ส่งข้อความแจ้งเตือน LINE
10. ล้างข้อมูลการเล่น
11. ปิดรอบ
12. ส่งรายงานสรุป

### ข้อมูลที่บันทึก
- **ชีท Results:** ผลลัพธ์ทั้งหมด
- **ชีท UsersBalance:** ยอดเงินคงเหลือ
- **ชีท Transactions:** รายการเงิน (ถ้าต้องการ)

### ข้อความที่ส่ง
1. **ส่วนตัวผู้ชนะ:** 🎉 ยินดีด้วย! คุณชนะ
2. **ส่วนตัวผู้แพ้:** 😔 เสียใจด้วย คุณแพ้
3. **กลุ่ม:** 📊 ผลการเล่น
4. **รายงาน:** 📊 สรุปผลการเล่น

## 🔄 ลำดับการใช้คำสั่ง

### ตัวอย่างการใช้งานทั้งหมด

```
1. Admin: :เริ่ม บั้งไฟ1
   System: ✅ เปิดรอบการเล่น: บั้งไฟ1

2. User1: ต 500
   System: ✅ บันทึกการเล่นสำเร็จ

3. User2: ล 500
   System: ✅ บันทึกการเล่นสำเร็จ

4. Admin: :หยุด
   System: ✅ รอบนี้ปิดการทายแล้ว

5. Admin: :สรุป บั้งไฟ1 50
   System: 📊 สรุปผลการเล่น
           🎉 ส่งข้อความส่วนตัวผู้ชนะ
           😔 ส่งข้อความส่วนตัวผู้แพ้
           📊 ส่งข้อความกลุ่ม
```

## 📊 สถานะรอบการเล่น

### State Diagram
```
CLOSED (เริ่มต้น)
   ↓
:เริ่ม
   ↓
OPEN (รับการเล่น)
   ↓
:หยุด
   ↓
CLOSED (ปิดการเล่น)
   ↓
:สรุป
   ↓
CALCULATING (คำนวณผล)
   ↓
CLOSED (เสร็จสิ้น)
```

## ⚠️ ข้อผิดพลาดที่อาจเกิดขึ้น

### 1. รูปแบบคำสั่งไม่ถูกต้อง
```
❌ คำสั่งไม่ถูกต้อง
```

### 2. ไม่พบการเล่น
```
❌ ไม่พบการเล่นสำหรับบั้งไฟ: บั้งไฟ1
```

### 3. เกิดข้อผิดพลาดในการประมวลผล
```
❌ เกิดข้อผิดพลาดในการประมวลผล
```

---

## 🔐 ข้อจำกัด

- เฉพาะแอดมินเท่านั้นที่สามารถใช้คำสั่งได้
- ต้องใช้รูปแบบคำสั่งที่ถูกต้อง
- ต้องมีการเล่นอย่างน้อย 1 คู่ก่อนสรุปผล
- ไม่สามารถสรุปผลซ้ำได้

---

## 💡 เคล็ดลับ

1. **ใช้ชื่อบั้งไฟที่ชัดเจน** เพื่อหลีกเลี่ยงความสับสน
2. **ตรวจสอบคะแนนก่อนสรุป** เพื่อให้แน่ใจว่าถูกต้อง
3. **รอให้ผู้เล่นส่งข้อความเล่นเสร็จ** ก่อนใช้คำสั่ง :หยุด
4. **ใช้คำสั่ง :สรุป ทีละครั้ง** เพื่อหลีกเลี่ยงข้อผิดพลาด
