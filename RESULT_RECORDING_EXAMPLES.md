## 📋 รูปแบบการบันทึก (Code)

### ข้อมูลที่ต้องอัปเดต

```javascript
// ข้อมูลที่ต้องอัปเดตในแถว
const updateData = {
  result: score, // Column I: ผลที่ออก
  resultWinLose: isDraw ? 'เสมอ' : 'ชนะ', // Column J: ผลแพ้ชนะ
  resultA: buildResultForPlayer(
    winner.displayName === bet1.displayName,
    isDraw,
    winner,
    loser
  ), // Column S: ผลลัพธ์ A
  resultB: buildResultForPlayer(
    loser.displayName === bet2.displayName,
    isDraw,
    winner,
    loser
  ) // Column T: ผลลัพธ์ B
};

// ฟังก์ชันสร้างข้อความผลลัพธ์
function buildResultForPlayer(isWinner, isDraw, winner, loser) {
  if (isDraw) {
    return `เสมอ หัก ${winner.fee} บาท`;
  }
  
  if (isWinner) {
    return `ชนะ ${winner.netAmount} บาท`;
  } else {
    return `แพ้ ${Math.abs(loser.netAmount)} บาท`;
  }
}
```

## 🔄 ขั้นตอนการบันทึก (SQL-like)

```sql
-- ค้นหาแถวที่จับคู่ไปแล้ว
SELECT * FROM Bets 
WHERE slipName = 'แอด' 
  AND matchedAuto = 'AUTO'
  AND resultWinLose IS NULL;

-- อัปเดตผลลัพธ์
UPDATE Bets 
SET 
  result = 340,
  resultWinLose = 'ชนะ',
  resultA = 'ชนะ 270 บาท',
  resultB = 'แพ้ 300 บาท'
WHERE rowIndex = 2;
```

## 📊 ตารางเปรียบเทียบ

| ประเด็น | ก่อนประกาศผล | หลังประกาศผล |
|--------|------------|-----------|
| Column I (ผลที่ออก) | (ว่างเปล่า) | 340 |
| Column J (ผลแพ้ชนะ) | (ว่างเปล่า) | ชนะ |
| Column S (ผลลัพธ์ A) | (ว่างเปล่า) | ชนะ 270 บาท |
| Column T (ผลลัพธ์ B) | (ว่างเปล่า) | แพ้ 300 บาท |
| ยอดเงิน User A | 0 | +270 บาท |
| ยอดเงิน User B | 0 | -300 บาท |

## 📌 สรุป

### ข้อมูลที่บันทึก
1. **Column I**: ผลที่ออก (Score)
2. **Column J**: ผลแพ้ชนะ (ชนะ/แพ้/เสมอ)
3. **Column S**: ผลลัพธ์ A (ชนะ/แพ้/เสมอ + จำนวนเงิน)
4. **Column T**: ผลลัพธ์ B (ชนะ/แพ้/เสมอ + จำนวนเงิน)

### ข้อมูลที่ไม่เปลี่ยน
- Column A-H: ข้อมูลการเล่น (ไม่เปลี่ยน)
- Column K-M: ข้อมูล User B (ไม่เปลี่ยน)
- Column U: MATCHED Auto (ไม่เปลี่ยน)

### ข้อมูลที่อัปเดตนอกชีท
- ยอดเงินผู้ชนะ: +Net Amount
- ยอดเงินผู้แพ้: -Gross Amount
