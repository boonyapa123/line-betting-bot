# สรุปการทำงานของระบบ

## ✅ ระบบทำงานแบบนี้ถูกต้อง

### 1️⃣ User A ส่งข้อความ
```
ไล่/360-450/20เป็ด
```
- ระบบ parse: ราคา 360-450, ฝั่ง ล, ยอดเงิน 20, บั้งไฟ เป็ด
- บันทึกลงชีท Bets

### 2️⃣ User B Reply
```
ต
```
- ระบบค้นหา pending bet ของ User A
- ใช้ข้อมูล User A: ราคา 360-450, ยอดเงิน 20, บั้งไฟ เป็ด
- คำนวณฝั่งตรงข้าม: ล → ต
- อัปเดตแถว User A ด้วยข้อมูล User B
- ส่ง confirmation: ✅ จับคู่เล่นสำเร็จ

### 3️⃣ User C ส่งข้อความ
```
ไล่/360-400/50/เป็ด
```
- ระบบ parse: ราคา 360-400, ฝั่ง ล, ยอดเงิน 50, บั้งไฟ เป็ด
- บันทึกลงชีท Bets (แถวใหม่)

### 4️⃣ User D Reply
```
ต
```
- ระบบค้นหา pending bet ของ User C
- ใช้ข้อมูล User C: ราคา 360-400, ยอดเงิน 50, บั้งไฟ เป็ด
- คำนวณฝั่งตรงข้าม: ล → ต
- อัปเดตแถว User C ด้วยข้อมูล User D
- ส่ง confirmation: ✅ จับคู่เล่นสำเร็จ

### 5️⃣ ประกาศผล
```
เป็ด 410
```

#### ข้อความ A (ไล่/360-450/20เป็ด):
- ราคา: 360-450
- ผล: 410 (อยู่ในช่วง)
- **ผลประกาศ: ⛔️ เสมอ**
- User A: ⛔️ หัก 5% = 1 บาท
- User B: ⛔️ หัก 5% = 1 บาท

#### ข้อความ B (ไล่/360-400/50/เป็ด):
- ราคา: 360-400
- ผล: 410 (สูงกว่าช่วง)
- **ผลประกาศ: ✅ ล ชนะ**
- User C: ✅ ชนะ ได้ 45 บาท
- User D: ❌ แพ้ เสีย 50 บาท

## 📋 ส่วนประกอบของระบบ

### ✅ ส่วนที่ทำงานถูกต้อง
1. **Message Parsing**: ถอดข้อความได้ถูกต้อง
2. **Slip Name Recording**: บันทึกชื่อบั้งไฟถูกต้อง (เฉพาะชื่อ ไม่มีราคา)
3. **Price Extraction**: ดึงราคาจากข้อความได้ถูกต้อง
4. **Reply Handling**: จับคู่ reply ได้ถูกต้อง
5. **Price Range Checking**: ตรวจสอบช่วงราคาได้ถูกต้อง
6. **Result Announcement**: ประกาศผลได้ถูกต้อง
7. **Fee Calculation**: คำนวณค่าธรรมเนียมได้ถูกต้อง

### 🎯 ลักษณะการทำงาน
- **แยกการประกาศผล**: แต่ละข้อความประกาศผลแยกกัน
- **ใช้ราคาต่างกัน**: ข้อความ A และ B มีราคาต่างกัน → ผลต่างกัน
- **ใช้บั้งไฟเดียวกัน**: ทั้งสองข้อความใช้บั้งไฟ "เป็ด"
- **ยอดเงินต่างกัน**: ข้อความ A 20 บาท, ข้อความ B 50 บาท
