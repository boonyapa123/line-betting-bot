# อธิบายการทำงานของทั้งสามชีท - Players, Transactions, UsersBalance

## 📊 ภาพรวม

ระบบใช้ 3 ชีทหลักเพื่อจัดการยอดเงิน:

```
┌─────────────────────────────────────────────────────────────┐
│                    ผู้เล่นส่งสลิป                            │
│                  (Slip2Go Webhook)                          │
└────────────────────────┬────────────────────────────────────┘
                         │
        ┌────────────────┴───────────────┐
        │                                │
        ▼                                ▼
   ┌─────────────┐            ┌──────────────────┐
   │  Players    │            │ Transactions     │
   │ (ข้อมูล)    │            │ (บันทึก)         │
   └─────────────┘            └──────────────────┘
        │                              │
        └──────────────┬───────────────┘
                       │
                       ▼
        ┌──────────────────────────┐
        │   UsersBalance           │
        │ (ยอดเงินปัจจุบัน)         │
        └──────────────────────────┘
```

---

## 🎯 ชีทแต่ละชีททำอะไร

### 1️⃣ ชีท Players - ข้อมูลผู้เล่นและยอดเงิน

**ทำหน้าที่:**
- เก็บข้อมูลผู้เล่นทั้งหมด
- บันทึกยอดเงินปัจจุบันของแต่ละคน
- เป็นแหล่งข้อมูลหลักของระบบ

**โครงสร้าง:**
```
| User ID | ชื่อ | เบอร์โทร | เลขบัญชี | ยอดเงิน |
|---------|------|---------|---------|---------|
| U123 | 💓Noon💓 | 0812345678 | 1234567890 | 5000 |
| U456 | นุช519 | 0898765432 | 0987654321 | 5000 |
| U789 | paa"BOY" | 0811111111 | 1111111111 | 5000 |
```

**ตัวอย่างข้อมูล:**
- 💓Noon💓 มีเงิน 5000 บาท
- นุช519 มีเงิน 5000 บาท
- paa"BOY" มีเงิน 5000 บาท

**เมื่อไหร่ที่ยอดเงินเปลี่ยน:**
1. ผู้เล่นส่งสลิป → ยอดเงินเพิ่ม
2. ผู้เล่นเล่น → ยอดเงินลด
3. ผู้เล่นชนะ → ยอดเงินเพิ่ม
4. ผู้เล่นแพ้ → ยอดเงินลด

---

### 2️⃣ ชีท Transactions - บันทึกรายการเงิน

**ทำหน้าที่:**
- บันทึกทุกการเปลี่ยนแปลงยอดเงิน
- เก็บประวัติการเงินของผู้เล่น
- ตรวจสอบความถูกต้องของยอดเงิน

**โครงสร้าง:**
```
| วันที่ | ชื่อผู้เล่น | ประเภท | จำนวนเงิน | สถานะ | ยอดเงินก่อน | ยอดเงินหลัง |
|--------|-----------|--------|---------|--------|-----------|-----------|
| 02/03/2569 | 💓Noon💓 | deposit | 20 | verified | 0 | 20 |
| 02/03/2569 | 💓Noon💓 | deposit | 10 | verified | 20 | 500 |
| 02/03/2569 | นุช519 | deposit | 500 | verified | 0 | 500 |
| 02/03/2569 | paa"BOY" | deposit | 400 | verified | 0 | 400 |
```

**ประเภทของรายการ:**
- **deposit** - ผู้เล่นส่งสลิป (เงินเข้า)
- **bet** - ผู้เล่นเล่น (เงินออก)
- **win** - ผู้เล่นชนะ (เงินเข้า)
- **lose** - ผู้เล่นแพ้ (เงินออก)

**ตัวอย่างการทำงาน:**
```
💓Noon💓 ส่งสลิป 20 บาท
→ Transactions: deposit | 20 | BalanceBefore: 0 | BalanceAfter: 20
→ Players: Balance = 20

💓Noon💓 ส่งสลิป 10 บาท
→ Transactions: deposit | 10 | BalanceBefore: 20 | BalanceAfter: 500
→ Players: Balance = 500
```

---

### 3️⃣ ชีท UsersBalance - ยอดเงินปัจจุบัน

**ทำหน้าที่:**
- เก็บยอดเงินปัจจุบันของผู้เล่น
- ใช้สำหรับตรวจสอบยอดเงินก่อนเล่น
- Sync จากชีท Players

**โครงสร้าง:**
```
| LineName | DisplayName | Balance |
|----------|-------------|---------|
| 💓Noon💓 | 💓Noon💓 | 5000 |
| นุช519 | นุช519 | 5000 |
| paa"BOY" | paa"BOY" | 5000 |
```

**ตัวอย่างข้อมูล:**
- 💓Noon💓 มีเงิน 5000 บาท (ตรวจสอบก่อนเล่น)
- นุช519 มีเงิน 5000 บาท (ตรวจสอบก่อนเล่น)
- paa"BOY" มีเงิน 5000 บาท (ตรวจสอบก่อนเล่น)

## 🔄 ความสัมพันธ์ระหว่างชีท

### ความสัมพันธ์แบบ Diagram

```
Players (แหล่งข้อมูลหลัก)
    ↓
    ├─→ Transactions (บันทึก)
    │
    └─→ UsersBalance (Sync)
            ↓
            └─→ BalanceCheckService (ตรวจสอบ)
```

### ความสัมพันธ์แบบ Flow

```
1. ผู้เล่นส่งสลิป
   ↓
   Players: Balance += Amount
   ↓
   Transactions: บันทึก (Type: deposit)
   ↓
   UsersBalance: Sync จาก Players

2. ผู้เล่นเล่น
   ↓
   UsersBalance: ตรวจสอบยอดเงิน
   ↓
   Players: Balance -= Amount
   ↓
   Transactions: บันทึก (Type: bet)
   ↓
   UsersBalance: Sync จาก Players

3. ผู้เล่นชนะ/แพ้
   ↓
   Players: Balance += NetAmount
   ↓
   Transactions: บันทึก (Type: win/lose)
   ↓
   UsersBalance: Sync จาก Players
```

## 🔗 ความสัมพันธ์ที่สำคัญ

### 1. Players ← → Transactions
- **Players** เป็นแหล่งข้อมูลหลัก
- **Transactions** บันทึกทุกการเปลี่ยนแปลง
- ทุกครั้งที่ Players เปลี่ยน → Transactions บันทึก

### 2. Players ← → UsersBalance
- **Players** มีข้อมูลครบถ้วน
- **UsersBalance** Sync จาก Players
- ทุกครั้งที่ Players เปลี่ยน → UsersBalance Sync

### 3. Transactions ← → UsersBalance
- **Transactions** บันทึกประวัติ
- **UsersBalance** ใช้ตรวจสอบยอดเงิน
- ยอดเงินใน UsersBalance ต้องตรงกับ BalanceAfter ใน Transactions

## ✅ ตรวจสอบความถูกต้อง

### ตรวจสอบ 1: Players = UsersBalance
```
Players: 💓Noon💓 = 5000 บาท
UsersBalance: 💓Noon💓 = 5000 บาท
✅ ตรงกัน
```

### ตรวจสอบ 2: Transactions BalanceAfter = Players Balance
```
Transactions: BalanceAfter = 5000 บาท
Players: Balance = 5000 บาท
✅ ตรงกัน
```

### ตรวจสอบ 3: Transactions มีบันทึกทุกการเปลี่ยนแปลง
```
Players: Balance เปลี่ยน
Transactions: มีบันทึก
✅ ถูกต้อง
```

## 💡 สรุป

| ชีท | ทำหน้าที่ | ข้อมูล | ความสัมพันธ์ |
|-----|---------|--------|-----------|
| **Players** | แหล่งข้อมูลหลัก | ข้อมูลผู้เล่น + ยอดเงิน | ← ได้รับจาก Slip2Go |
| **Transactions** | บันทึกประวัติ | รายการเงิน + ประวัติ | ← ได้รับจาก Players |
| **UsersBalance** | ตรวจสอบยอดเงิน | ยอดเงินปัจจุบัน | ← Sync จาก Players |

**ทำงานสัมพันธ์กัน:**
1. Players เป็นแหล่งข้อมูลหลัก
2. Transactions บันทึกทุกการเปลี่ยนแปลง
3. UsersBalance Sync ยอดเงินปัจจุบัน
4. ทั้งสามชีทต้องตรงกัน
