### ทำการ:

- ตรวจสอบว่าบั้งไฟตรงกันหรือไม่

### ตัวอย่าง:

## 3️⃣ **ขั้นตอนที่สาม: คำนวณผลลัพธ์**

### ตรวจสอบว่าคะแนนอยู่ในเกณฑ์ของใครบ้าง:

```
User A (ล): 350-370
  - คะแนน 370 ✅ อยู่ในเกณฑ์

User B (ย): 350-410
  - คะแนน 370 ✅ อยู่ในเกณฑ์

ผลลัพธ์: ⛔️ ออกกลาง (ทั้งสองฝั่งอยู่ในเกณฑ์)
```

### คำนวณค่าธรรมเนียม:
```
ยอดเงิน: 300 บาท
ค่าธรรมเนียม: 300 × 5% = 15 บาท (ออกกลาง)

User A: -15 บาท (หัก 5%)
User B: -15 บาท (หัก 5%)
```

### ตรวจสอบ:
ดูใน `index.js` บรรทัด 415-462 (คำนวณ userAWinnings และ userBWinnings)

| T | ผลลัพธ์ B | ผลและจำนวน | เสมอ หัก 15 บาท |

## ⚠️ **ปัญหาที่อาจเกิดขึ้น**

### ❌ ไม่บันทึกผลลัพธ์
**สาเหตุ:**
- ไม่พบแถวที่จับคู่
- ชื่อบั้งไฟไม่ตรงกัน
- ข้อมูล User B ไม่สมบูรณ์

**วิธีแก้:**
```bash
node verify-result-recording.js
```
ดูว่าพบแถวที่จับคู่หรือไม่

### ❌ ผลลัพธ์ไม่ถูกต้อง
**สาเหตุ:**
- คำนวณค่าธรรมเนียมผิด
- สถานะผลลัพธ์ผิด

**วิธีแก้:**
ตรวจสอบการคำนวณใน `index.js` บรรทัด 415-462

### ❌ ไม่ส่งแจ้งเตือน
**สาเหตุ:**
- ไม่มี User ID
- Token LINE ไม่ถูกต้อง

**วิธีแก้:**
