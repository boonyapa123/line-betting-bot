# ระบบจับคู่และการบันทึกลงชีท

## 1. ขั้นตอนการทำงาน

### ขั้นตอนที่ 1: User A ส่งข้อความเดิมพัน
```
User A: "310-320 ย 20 ฟ้า"
```

**ระบบทำ:**
1. Parse ข้อความ → ได้ `parsedBet`:
   - `price`: "310-320"
   - `sideCode`: "ย" (ยั้ง)
   - `amount`: 20
   - `slipName`: "ฟ้า"

2. ตรวจสอบยอดเงิน User A

3. **บันทึกลงชีท Bets** (แถวใหม่):
   - Column A (Timestamp): 13/03/2569 22:19:13
   - Column B (User A ID): Uc2a009fe53d51946657363bdbb7d1374
   - Column C (User A Name): 💓Noon💓
   - Column D (Message A): "310-320 ย 20 ฟ้า"
   - Column E (Slip Name): ฟ้า
   - Column F (Side A): ย
   - Column G (Amount): 20
   - Column H-U: ว่างเปล่า (รอ User B)

4. ส่งข้อความยืนยัน: "✅ บันทึกการเดิมพันสำเร็จ ⏳ รอการจับคู่..."

---

### ขั้นตอนที่ 2: ระบบค้นหาคู่อัตโนมัติ

**เงื่อนไขการจับคู่ (Price Range Matching):**

```javascript
// ต้องตรงกันทั้งหมด:
1. บั้งไฟเดียวกัน (slipName)
   ✓ ฟ้า = ฟ้า

2. ฝั่งตรงข้าม (sideCode)
   ✓ ย ↔ ล (ยั้ง ↔ ไล่)

3. ช่วงราคาเดียวกัน (price)
   ✓ 310-320 = 310-320

4. ทั้งคู่ต้องมีราคา (Direct Method)
   ✓ ทั้งคู่มี price

5. ยังไม่ matched (status ≠ 'MATCHED')
   ✓ ยังรอการจับคู่
```

**ตัวอย่างการจับคู่:**

```
User A: "310-320 ย 20 ฟ้า"  → ย (ยั้ง), 310-320, 20 บาท
User B: "310-320 ล 20 ฟ้า"  → ล (ไล่), 310-320, 20 บาท

✓ ตรงกัน! → จับคู่สำเร็จ
```

**ตัวอย่างไม่จับคู่:**

```
User A: "310-320 ย 20 ฟ้า"
User B: "310-320 ล 30 ฟ้า"  ← ยอดเงินต่างกัน (20 ≠ 30)
❌ ไม่จับคู่ (ใช้ยอดน้อยกว่า = 20)

User A: "310-320 ย 20 ฟ้า"
User B: "300-320 ล 20 ฟ้า"  ← ช่วงราคาต่างกัน
❌ ไม่จับคู่

User A: "310-320 ย 20 ฟ้า"
User B: "ชล 20 ฟ้า"         ← ไม่มีราคา (Method 1)
❌ ไม่จับคู่
```

### ขั้นตอนที่ 3: User B ส่งข้อความตอบ

**วิธีที่ 1: Reply ด้วย Direct Method**
```
User B: "310-320 ล 20 ฟ้า"
```

**วิธีที่ 2: Reply ด้วย Simple Reply**
```
User B: "ต"  (ระบบจะใช้ข้อมูล User A และคำนวณฝั่งตรงข้าม)
```

### ขั้นตอนที่ 4: ระบบจับคู่และอัปเดตชีท

**ถ้าจับคู่สำเร็จ:**

1. **อัปเดตแถวเดิม** (ไม่เพิ่มแถวใหม่):
   - Column H (User B ID): Uc2a009fe53d51946657363bdbb7d1374
   - Column I (User B Name): 💓Noon💓
   - Column J (Side B): ล
   - Column K (Amount B): 20
   - Column L (Price B): 310-320
   - Column M (Status): MATCHED

2. ส่งข้อความยืนยัน:
   ```
   ✅ จับคู่เล่นสำเร็จ
   
   🎆 บั้งไฟ: ฟ้า
   💹 ราคา: 310-320
   
   👤 paa"BOY"
      ฝั่ง: ล
      ยอดเงิน: 20 บาท
   
   👤 💓Noon💓
      ฝั่ง: ย
      ยอดเงิน: 20 บาท
   
   ⏳ รอการประกาศผล...
   ```

**ถ้าไม่จับคู่:**

1. **บันทึกแถวใหม่** (User B เป็น User A ของแถวใหม่):
   - Column A (Timestamp): 13/03/2569 22:19:13
   - Column B (User B ID): Uc2a009fe53d51946657363bdbb7d1374
   - Column C (User B Name): 💓Noon💓
   - Column D (Message): "310-320 ล 20 ฟ้า"
   - Column E (Slip Name): ฟ้า
   - Column F (Side A): ล
   - Column G (Amount): 20
   - Column H-U: ว่างเปล่า (รอคู่ใหม่)

2. ส่งข้อความแจ้ง:
   ```
   ⚠️ ไม่สามารถจับคู่กับการเดิมพันเดิมได้
   บันทึกเป็นการเดิมพันใหม่แทน
   
   🎆 บั้งไฟ: ฟ้า
   💹 ราคา: 310-320
   💰 ยอดเงิน: 20 บาท
   
   ⏳ รอการจับคู่...
   ```

## 3. ขั้นตอนการประกาศผล

### ขั้นตอนที่ 1: Admin ประกาศผล
```
:สรุป ฟ้า 315
```

### ขั้นตอนที่ 2: ระบบคำนวณผล
1. ค้นหาเบตทั้งหมดของบั้งไฟ "ฟ้า"
2. สำหรับแต่ละเบต:
   - ตรวจสอบว่า 315 อยู่ในช่วงราคา User A หรือ User B
   - คำนวณผลชนะ/แพ้
   - อัปเดตยอดเงิน

### ขั้นตอนที่ 3: อัปเดตชีท
- Column N (Result): 315
- Column O (Result Win/Lose A): ชนะ/แพ้
- Column P (Result Win/Lose B): ชนะ/แพ้
- Column U (Result A): ยอดเงินที่ได้

## 4. ตัวอย่างการทำงาน

### สถานการณ์: User A และ User B เดิมพันบั้งไฟ "ฟ้า"

**ข้อมูลเบต:**
```
User A (paa"BOY"): ล (ไล่), ช่วง 310-320, 20 บาท
User B (💓Noon💓): ย (ยั้ง), ช่วง 310-320, 20 บาท
```

**ประกาศผล: 315**

**ผลการคำนวณ:**
- 315 อยู่ในช่วง 310-320 ✓
- User A (ล/ไล่) ชนะ → ได้ 20 บาท
- User B (ย/ยั้ง) แพ้ → เสีย 20 บาท

**ชีท Bets ถูกอัปเดต:**
- Column N: 315
- Column O: ชนะ
- Column P: แพ้
- Column U: 20

---

## 5. ข้อสังเกต

### ข้อมูลที่บันทึกลงชีท
- **ครั้งแรก**: เมื่อ User A ส่งข้อความ (บันทึกแถวใหม่)
- **ครั้งที่สอง**: เมื่อจับคู่สำเร็จ (อัปเดตแถวเดิม)
- **ครั้งที่สาม**: เมื่อประกาศผล (อัปเดตผลและยอดเงิน)

### ปัญหาปัจจุบัน
- `recordBet` ไม่ได้รับข้อมูล `parsedBet` ที่ถูกต้อง
- ล็อกแสดง `Message: undefined undefined`
- ต้องตรวจสอบว่า `betData` ถูกส่งไปอย่างไร
