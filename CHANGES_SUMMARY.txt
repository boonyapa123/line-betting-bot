================================================================================
KEY FEATURES
================================================================================

1. PENDING BALANCE CHECKING
   - Calculates total pending bets (matched but no result)
   - Checks if available balance >= new bet amount
   - Available balance = current balance - pending amount

2. AUTOMATIC DEDUCTION
   - Deducts bet amount immediately when pair is matched
   - Uses minimum amount: betAmount = Math.min(bet1.amount, bet2.amount)
   - Updates balance in Google Sheets

3. DETAILED NOTIFICATIONS
   - Shows current balance
   - Shows pending amount
   - Shows available balance
   - Shows shortfall if insufficient

4. PENDING BETS TRACKING
   - Lists all pending bets with details
   - Shows opponent and amount
   - Shows timestamp

================================================================================
DATA FLOW CHANGES
================================================================================

BEFORE:
  1. PARSING → 2. VALIDATION → 3. RECORDING → 4. PAIRING → 5. SETTLEMENT → 6. BALANCE UPDATE

AFTER:
  1. PARSING → 2. VALIDATION → 3. BALANCE CHECK (NEW) → 4. RECORDING → 5. PAIRING → 6. DEDUCT (NEW) → 7. SETTLEMENT → 8. BALANCE UPDATE

================================================================================
GOOGLE SHEETS SCHEMA
================================================================================

BETS SHEET (Columns A-N):
  A: Timestamp
  B: User A ID
  C: ชื่อ User A
  D: ข้อความ A
  E: ชื่อบั้งไฟ
  F: รายการเล่น
  G: ยอดเงิน A
  H: ยอดเงิน B (empty = not matched)
  I: แสดงผล (empty = no result)
  J: แสดงผลชนะ
  K: User B ID
  L: ชื่อ User B
  M: รายการเล่น B
  N: ผลลัพธ์สุดท้าย

USERSBALANCE SHEET (Columns A-C):
  A: User ID
  B: Display Name
  C: Balance

TRANSACTIONS SHEET (Columns A-G):
  A: Timestamp
  B: Player Name
  C: Transaction Type
  D: Amount
  E: Previous Balance
  F: New Balance
  G: Slip Name

================================================================================
PENDING BALANCE CALCULATION
================================================================================

Checks Bets sheet for rows where:
  1. Column H (ยอดเงิน B) has value = MATCHED
  2. Column I (แสดงผล) is empty = NO RESULT

Then sums the amounts for the player:
  - If User A: sum Column G (ยอดเงิน A)
  - If User B: sum Column H (ยอดเงิน B)

EXAMPLE SCENARIO
================================================================================

User A: Balance 1000, Pending 500
User B: Balance 800, Pending 0

Scenario 1: User A wants to bet 600
  - Available balance: 1000 - 500 = 500
  - Required: 600
  - Result: INSUFFICIENT ❌
  - Notification sent with details

Scenario 2: User A wants to bet 400
  - Available balance: 1000 - 500 = 500
  - Required: 400
  - Result: SUFFICIENT ✅
  - Bet recorded
  - Pair matched (use 300)
  - Deduct 300 from both players
  - User A: 1000 → 700
  - User B: 800 → 500
