# 📢 คู่มือการประกาศผล (Result Announcement Guide)

## 🎯 ภาพรวม

ระบบการประกาศผลมี 3 ขั้นตอนหลัก:

1. **ประกาศผล** - ส่งคำสั่งประกาศผล
2. **ตรวจสอบผล** - ระบบตรวจสอบว่าใครชนะ/แพ้
3. **บันทึกผล** - ระบบบันทึกผลลงชีท และแจ้งเตือนผู้เล่น

---

## 📝 ขั้นตอนที่ 1: ประกาศผล

### รูปแบบคำสั่ง

```
:สรุป [ชื่อบั้งไฟ] [คะแนน]
```

### ตัวอย่าง

```
:สรุป ฟ้า 370
:สรุป แอดเทวดา 325
:สรุป เป็ด 450
```

### ส่วนประกอบ

| ส่วน | ตัวอย่าง | คำอธิบาย |
|------|---------|---------|
| `:สรุป` | `:สรุป` | คำสั่งประกาศผล (ต้องมี) |
| ชื่อบั้งไฟ | `ฟ้า` | ชื่อบั้งไฟที่ประกาศผล |
| คะแนน | `370` | คะแนนที่ออก (ตัวเลข) |

### ข้อสังเกต

- ✅ ต้องใช้คำสั่ง `:สรุป` (ขึ้นต้นด้วย colon)
- ✅ ชื่อบั้งไฟต้องตรงกับที่ผู้เล่นใช้
- ✅ คะแนนต้องเป็นตัวเลข
- ❌ ห้ามใช้ตัวอักษรพิเศษในคะแนน
- ❌ ห้ามใช้ช่องว่างเกินจำเป็น

---

## 🔍 ขั้นตอนที่ 2: ตรวจสอบผล

### ระบบทำงานอย่างไร

เมื่อคุณประกาศผล `:สรุป ฟ้า 370` ระบบจะ:

#### 1️⃣ ค้นหาการเล่นทั้งหมดของบั้งไฟ "ฟ้า"

```
ค้นหาจากชีท Bets:
- Column E (ชื่อบั้งไฟ) = "ฟ้า"
- Column H (ยอดเงิน B) ≠ ว่าง (เฉพาะคู่ที่จับได้)
```

#### 2️⃣ ตรวจสอบผลลัพธ์ของแต่ละคู่

**ตัวอย่าง 1: ผลออกในช่วง (เสมอ)**
```
User A: ไล่/325-340/100ฟ้า (ล = สูง)
User B: reply ล
ผลออก: 330

ตรวจสอบ:
- 330 อยู่ในช่วง 325-340? ✅ ใช่
- ผลลัพธ์: ⛔️ เสมอ (ทั้งสองฝั่งเสีย 5%)
```

**ตัวอย่าง 2: ผลออกต่ำกว่าช่วง (ฝ่าย ย ชนะ)**
```
User A: ย/310-320/20ฟ้า (ย = ต่ำ)
User B: reply ต
ผลออก: 305

ตรวจสอบ:
- 305 < 310 (ต่ำกว่าช่วง)? ✅ ใช่
- User A เป็น ย (ต่ำ)? ✅ ใช่
- ผลลัพธ์: ✅ User A ชนะ, ❌ User B แพ้
```

**ตัวอย่าง 3: ผลออกสูงกว่าช่วง (ฝ่าย ล ชนะ)**
```
User A: ล/300-340/30ฟ้า (ล = สูง)
User B: reply ล
ผลออก: 350

ตรวจสอบ:
- 350 > 340 (สูงกว่าช่วง)? ✅ ใช่
- User A เป็น ล (สูง)? ✅ ใช่
- ผลลัพธ์: ✅ User A ชนะ, ❌ User B แพ้
```

### ตรรกะการตรวจสอบ

#### 📊 ตารางการตรวจสอบผล

| สถานการณ์ | ผลลัพธ์ | หมายเหตุ |
|---------|--------|--------|
| ผลออกในช่วง | ⛔️ เสมอ | ทั้งสองฝั่งเสีย 5% |
| ผลออกต่ำกว่าช่วง + User A เป็น ย | ✅ A ชนะ | A ได้รับ 90%, B เสีย 100% |
| ผลออกต่ำกว่าช่วง + User A เป็น ล | ❌ A แพ้ | A เสีย 100%, B ได้รับ 90% |
| ผลออกสูงกว่าช่วง + User A เป็น ล | ✅ A ชนะ | A ได้รับ 90%, B เสีย 100% |
| ผลออกสูงกว่าช่วง + User A เป็น ย | ❌ A แพ้ | A เสีย 100%, B ได้รับ 90% |

#### 🎯 ความหมายของ ย/ล

- **ย** = ต่ำ (ยิ่งต่ำยิ่งดี) → ชนะเมื่อผลต่ำกว่าช่วง
- **ล** = สูง (ลูกศรขึ้น = สูง) → ชนะเมื่อผลสูงกว่าช่วง

## 💾 ขั้นตอนที่ 3: บันทึกผล

### ระบบบันทึกอะไรบ้าง

เมื่อตรวจสอบผลเสร็จ ระบบจะบันทึก:

#### 1️⃣ ผลลัพธ์ (Column I, J, K)

```
Column I (ผลที่ออก):     370
Column J (ผลแพ้ชนะ A):   ✅ หรือ ❌ หรือ ⛔️
Column K (ผลแพ้ชนะ B):   ❌ หรือ ✅ หรือ ⛔️
```

#### 2️⃣ ยอดเงิน (Column S, T)

```
Column S (ยอดเงิน A):    +27 หรือ -30 หรือ -2
Column T (ยอดเงิน B):    -30 หรือ +27 หรือ -2
```

### ตัวอย่างการบันทึก

**ตัวอย่าง 1: User A ชนะ**
```
ข้อมูลการเล่น:
- User A: ไล่/325-340/100ฟ้า
- User B: reply ล
- ผลออก: 350

บันทึกผล:
- Column I: 350
- Column J: ✅ (User A ชนะ)
- Column K: ❌ (User B แพ้)
- Column S: +90 (User A ได้รับ 100 - 10%)
- Column T: -100 (User B เสีย)
```

**ตัวอย่าง 2: เสมอ**
```
ข้อมูลการเล่น:
- User A: ล/325-340/100ฟ้า
- User B: reply ล
- ผลออก: 330

บันทึกผล:
- Column I: 330
- Column J: ⛔️ (เสมอ)
- Column K: ⛔️ (เสมอ)
- Column S: -5 (User A เสีย 100 × 5%)
- Column T: -5 (User B เสีย 100 × 5%)
```

### ค่าธรรมเนียม

| สถานการณ์ | ค่าธรรมเนียม | ตัวอย่าง (100 บาท) |
|---------|-----------|-----------------|
| ชนะ-แพ้ | 10% | ผู้ชนะ: +90, ผู้แพ้: -100 |
| เสมอ | 5% | ทั้งสอง: -5 |

## 📱 ตัวอย่างการใช้งานจริง

### สถานการณ์ 1: ประกาศผลฟ้า 370

```
Admin: :สรุป ฟ้า 370

ระบบทำงาน:
1. ค้นหาการเล่นบั้งไฟ "ฟ้า"
   - พบ 3 คู่

2. ตรวจสอบแต่ละคู่:
   
   คู่ที่ 1:
   - User A: ไล่/325-340/100ฟ้า
   - User B: reply ล
   - ผลออก: 370 > 340 (สูงกว่าช่วง)
   - User A เป็น ล (สูง) → ✅ A ชนะ
   
   คู่ที่ 2:
   - User A: ย/360-380/50ฟ้า
   - User B: reply ย
   - ผลออก: 370 (อยู่ในช่วง)
   - ⛔️ เสมอ
   
   คู่ที่ 3:
   - User A: ล/300-350/200ฟ้า
   - User B: reply ล
   - ผลออก: 370 > 350 (สูงกว่าช่วง)
   - User A เป็น ล (สูง) → ✅ A ชนะ

3. บันทึกผล:
   - คู่ที่ 1: Column I=370, J=✅, K=❌, S=+90, T=-100
   - คู่ที่ 2: Column I=370, J=⛔️, K=⛔️, S=-2.5, T=-2.5
   - คู่ที่ 3: Column I=370, J=✅, K=❌, S=+180, T=-200

4. แจ้งเตือน LINE:
   - ส่วนตัว: ส่งข้อความให้ผู้ชนะและผู้แพ้
   - กลุ่ม: ประกาศผลรวม
```

## ⚠️ ข้อผิดพลาดที่พบบ่อย

### ❌ ข้อผิดพลาด 1: ชื่อบั้งไฟไม่ตรงกัน

```
ผู้เล่นใช้: ฟ้าหลังฝน
Admin ประกาศ: :สรุป ฟ้า 370

ผลลัพธ์: ❌ ไม่พบการเล่น
```

**วิธีแก้:** ใช้ชื่อบั้งไฟเดียวกับที่ผู้เล่นใช้

### ❌ ข้อผิดพลาด 2: ลืมคำสั่ง :สรุป

```
Admin พิมพ์: สรุป ฟ้า 370

ผลลัพธ์: ❌ ระบบไม่รู้ว่าเป็นคำสั่ง
```

**วิธีแก้:** ต้องขึ้นต้นด้วย `:สรุป`

### ❌ ข้อผิดพลาด 3: คะแนนไม่ใช่ตัวเลข

```
Admin พิมพ์: :สรุป ฟ้า สามร้อยเจ็ด

ผลลัพธ์: ❌ ระบบไม่สามารถแยกคะแนน
```

**วิธีแก้:** ใช้ตัวเลข เช่น `370`

## 🔄 ขั้นตอนการประกาศผลแบบสมบูรณ์

```
1. Admin ประกาศผล
   ↓
2. ระบบค้นหาการเล่นทั้งหมด
   ↓
3. ระบบตรวจสอบผลแต่ละคู่
   ↓
4. ระบบคำนวณค่าธรรมเนียม
   ↓
5. ระบบบันทึกผลลงชีท
   ↓
6. ระบบอัปเดตยอดเงินผู้เล่น
   ↓
7. ระบบแจ้งเตือน LINE (ส่วนตัว + กลุ่ม)
   ↓
8. ระบบล้างข้อมูลรอบนี้
   ↓
9. ปิดรอบ
```
