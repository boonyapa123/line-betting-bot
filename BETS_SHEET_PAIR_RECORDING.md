# การบันทึกข้อมูลฝั่งคู่เล่น - Pair Recording in Bets Sheet

## 📊 ภาพรวม

เมื่อผู้เล่นคนที่สองตอบกลับ ระบบต้องบันทึกข้อมูลของฝั่งคู่เล่นลงในคอลัมน์ H-M

---

## 🔄 ขั้นตอนการบันทึก

### Step 1: ผู้เล่น A ลงเล่นตรง

```
Input: "ลูกชายภูน้อย ชล 150"

Bets Sheet (Row 2):
| A | B | C | D | E | F | G | H | I | J | K | L | M | N |
| 2024-03-02T14:56:01.477Z | U2fbb940e366149fc8dbdf92b92a81b60 | 💓Noon💓 | ลูกชายภูน้อย ชล 150 | ลูกชายภูน้อย | ชล | 150 | | | | | | | |
```

**บันทึก:**
- A: Timestamp
- B: User A ID
- C: ชื่อ User A
- D: ข้อความ A
- E: ชื่อบั้งไฟ
- F: รายการเล่น (ฝั่ง)
- G: ยอดเงิน
- H-N: ว่างเปล่า (รอการจับคู่)

---

### Step 2: ผู้เล่น B ตอบกลับ

```
Input: "ต" (reply)

Bets Sheet (Row 2 - อัปเดต):
| A | B | C | D | E | F | G | H | I | J | K | L | M | N |
| 2024-03-02T14:56:01.477Z | U2fbb940e366149fc8dbdf92b92a81b60 | 💓Noon💓 | ลูกชายภูน้อย ชล 150 | ลูกชายภูน้อย | ชล | 150 | 150 | | | Ubb4a196cd0dc0c3ca6211a9bafecb513 | Pam Yuthida | ชถ | |
```

**อัปเดต:**
- H: ยอดเงิน B = 150 (เท่ากับ User A)
- K: User B ID = Ubb4a196cd0dc0c3ca6211a9bafecb513
- L: ชื่อ User B = Pam Yuthida
- M: รายการเล่น B = ชถ (ฝั่งตรงข้าม)
- I, J, N: ว่างเปล่า (รอสรุปผล)

---

## 📝 รายละเอียดการบันทึกฝั่งคู่เล่น

### H: ยอดเงิน B (จำนวนเงิน User B)
**ทำหน้าที่:** บันทึกจำนวนเงินที่ User B เล่น

**ข้อมูล:** ตัวเลข (บาท)
```
150 (เท่ากับ User A)
```

**หมายเหตุ:**
- ต้องเท่ากับ G (ยอดเงิน A)
- ถ้าต่างกัน ให้ใช้ยอดน้อยกว่า

---

### K: User B ID (ID ผู้เล่น B)
**ทำหน้าที่:** ระบุตัวตนของผู้เล่น B

**ข้อมูล:** LINE User ID
```
Ubb4a196cd0dc0c3ca6211a9bafecb513
```

---

### L: ชื่อ User B (ชื่อผู้เล่น B)
**ทำหน้าที่:** แสดงชื่อผู้เล่น B

**ข้อมูล:** ชื่อที่ LINE แสดง
```
Pam Yuthida
```

---

### M: รายการเล่น B (ฝั่ง User B)
**ทำหน้าที่:** บันทึกฝั่งที่ User B เล่น

**ข้อมูล:** ฝั่งตรงข้าม
```
ชถ (ถ้า User A เล่น ชล)
ชล (ถ้า User A เล่น ชถ)
```

**หมายเหตุ:**
- ต้องตรงข้ามกับ F (รายการเล่น A)
- ชล ↔ ชถ
- ล ↔ ย

---

## 📋 ตารางเปรียบเทียบ

### Reply Method

| คอลัมน์ | User A | User B |
|--------|--------|--------|
| **D** | ข้อความ A | (ไม่มี) |
| **E** | ชื่อบั้งไฟ | ชื่อบั้งไฟ (เดียวกัน) |
| **F** | ชล | ชถ (ตรงข้าม) |
| **G** | 150 | 150 (เท่ากัน) |
| **H** | (ว่าง) | 150 |
| **K** | (ว่าง) | User B ID |
| **L** | (ว่าง) | ชื่อ User B |
| **M** | (ว่าง) | ชถ |

## ✅ ตรวจสอบการบันทึก

### ตรวจสอบ 1: ข้อมูลครบถ้วน
```
✅ H: ยอดเงิน B - มี
✅ K: User B ID - มี
✅ L: ชื่อ User B - มี
✅ M: รายการเล่น B - มี
```

### ตรวจสอบ 2: ข้อมูลถูกต้อง
```
✅ H = G (Reply) หรือ H ≤ G (Direct)
✅ M ตรงข้ามกับ F
✅ E เดียวกัน
```

### ตรวจสอบ 3: ข้อมูลสอดคล้อง
```
✅ User A ≠ User B
✅ Side A ≠ Side B
✅ Amount A = Amount B (Reply) หรือ Amount A ≥ Amount B (Direct)
```
