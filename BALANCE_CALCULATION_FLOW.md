# ระบบคำนวนยอดเงินคงเหลือระหว่างเล่น

## 1️⃣ ขั้นตอนการคำนวนยอดเงิน

### ที่มา: `pendingBalanceService.js` และ `balanceUpdateService.js`

ระบบแบ่งการคำนวนยอดเงินออกเป็น 2 ช่วง:
1. **ระหว่างเล่น** (เล่นแล้ว แต่ผลยังไม่ออก)
2. **หลังผลออก** (ประกาศผลแล้ว)

---

## 2️⃣ ระหว่างเล่น (Pending Balance)

### 📌 สถานะ: MATCHED แต่ยังไม่มีผล

**ตัวอย่าง:**
```
User A: "ลูกชายภูน้อย ชล 150"
User B: "ลูกชายภูน้อย ชถ 100"

ชีท Bets:
  G (เงิน A): 150 บาท
  H (เงิน B): 100 บาท
  I (ผลลัพธ์): (ว่างเปล่า) ← ยังไม่มีผล
  
→ สถานะ: MATCHED แต่ยังไม่มีผล
```

---

### 🔍 การคำนวนยอดเงินค้าง

#### ที่มา: `pendingBalanceService.js` - `getPendingAmount()`

**ขั้นตอน:**
1. ดึงข้อมูลทั้งหมดจากชีท Bets
2. ค้นหาแถวที่ MATCHED แต่ยังไม่มีผล (H มีค่า, I ว่างเปล่า)
3. รวมยอดเงินทั้งหมดของผู้เล่น

**ตัวอย่าง:**
```
User A มีการเล่น 3 ครั้ง:
  1. ลูกชายภูน้อย ชล 150 → MATCHED, ยังไม่มีผล
  2. ลูกชายภูน้อย ชล 100 → MATCHED, ยังไม่มีผล
  3. ลูกชายภูน้อย ชล 50  → ยังไม่จับคู่

เงินค้าง = 150 + 100 = 250 บาท
```

---

### 💰 ยอดเงินที่สามารถใช้ได้

#### ที่มา: `pendingBalanceService.js` - `checkSufficientBalance()`

**สูตร:**
```
ยอดเงินที่สามารถใช้ได้ = ยอดเงินคงเหลือ - เงินค้าง
```

**ตัวอย่าง:**
```
ยอดเงินคงเหลือ: 1000 บาท
เงินค้าง: 250 บาท
ยอดเงินที่สามารถใช้ได้ = 1000 - 250 = 750 บาท

ถ้าต้องการเดิมพัน 100 บาท:
  750 >= 100 ✓ เพียงพอ

ถ้าต้องการเดิมพัน 800 บาท:
  750 >= 800 ✗ ไม่เพียงพอ
  ขาด: 800 - 750 = 50 บาท
```

---

### ⚠️ ตรวจสอบเงินก่อนเดิมพัน

**ขั้นตอน:**
1. ดึงยอดเงินคงเหลือปัจจุบัน
2. คำนวนเงินค้าง
3. คำนวนยอดเงินที่สามารถใช้ได้
4. ตรวจสอบว่าเพียงพอหรือไม่

**โค้ด:**
```javascript
const checkResult = await pendingBalanceService.checkSufficientBalance(
  displayName,
  currentBalance,
  newBetAmount
);

if (!checkResult.isSufficient) {
  // ยอดเงินไม่พอ
  console.log(`ขาด: ${checkResult.shortfall} บาท`);
}
```

## 3️⃣ หลังผลออก (Update Balance)

### 📌 สถานะ: ผลลัพธ์ออกแล้ว

**ตัวอย่าง:**
```
ชีท Bets:
  G (เงิน A): 150 บาท
  H (เงิน B): 100 บาท
  I (ผลลัพธ์): ชล ← ผลลัพธ์ออกแล้ว
  J (ผู้ชนะ): User A
  
→ สถานะ: ผลลัพธ์ออกแล้ว
```

---

### 🔄 การอัปเดตยอดเงิน

#### ที่มา: `balanceUpdateService.js` - `updateBalancesForResult()`

**ขั้นตอน:**
1. คำนวนผลลัพธ์ (ผู้ชนะ, ผู้แพ้, ค่าธรรมเนียม)
2. อัปเดตยอดเงินของผู้ชนะ
3. อัปเดตยอดเงินของผู้แพ้
4. บันทึกธุรกรรม

**ตัวอย่าง:**
```
ผลลัพธ์:
  ผู้ชนะ: User A
  ผู้แพ้: User B
  ยอดเงิน: 100 บาท
  ค่าธรรมเนียม: 10 บาท (10%)

User A:
  ยอดเงินเดิม: 1000 บาท
  ได้รับ: 100 - 10 = 90 บาท
  ยอดเงินใหม่: 1000 + 90 = 1090 บาท

User B:
  ยอดเงินเดิม: 800 บาท
  เสีย: 100 บาท
  ยอดเงินใหม่: 800 - 100 = 700 บาท
```

---

## 4️⃣ ตัวอย่างการไหลของข้อมูล

### สถานการณ์: User A เดิมพัน 150 บาท, User B ตอบรับ 100 บาท

```
Step 1: User A เดิมพัน
  ยอดเงิน A: 1000 บาท
  ยอดเงินค้าง: 0 บาท
  ยอดเงินที่สามารถใช้ได้: 1000 บาท
  
  ตรวจสอบ: 1000 >= 150 ✓
  → บันทึกการเล่น

Step 2: User B ตอบรับ
  ยอดเงิน B: 800 บาท
  ยอดเงินค้าง: 0 บาท
  ยอดเงินที่สามารถใช้ได้: 800 บาท
  
  ตรวจสอบ: 800 >= 100 ✓
  → จับคู่สำเร็จ

Step 3: ระหว่างรอผล
  User A:
    ยอดเงินคงเหลือ: 1000 บาท
    เงินค้าง: 150 บาท
    ยอดเงินที่สามารถใช้ได้: 850 บาท
    
  User B:
    ยอดเงินคงเหลือ: 800 บาท
    เงินค้าง: 100 บาท
    ยอดเงินที่สามารถใช้ได้: 700 บาท

Step 4: ประกาศผล (User A ชนะ)
  ยอดเงิน: 100 บาท
  ค่าธรรมเนียม: 10 บาท
  
  User A:
    ยอดเงินเดิม: 1000 บาท
    ได้รับ: 90 บาท
    ยอดเงินใหม่: 1090 บาท
    
  User B:
    ยอดเงินเดิม: 800 บาท
    เสีย: 100 บาท
    ยอดเงินใหม่: 700 บาท

Step 5: หลังอัปเดต
  User A:
    ยอดเงินคงเหลือ: 1090 บาท
    เงินค้าง: 0 บาท
    ยอดเงินที่สามารถใช้ได้: 1090 บาท
    
  User B:
    ยอดเงินคงเหลือ: 700 บาท
    เงินค้าง: 0 บาท
    ยอดเงินที่สามารถใช้ได้: 700 บาท
```

## 5️⃣ ตรวจสอบเงินค้าง

### ที่มา: `pendingBalanceService.js` - `getPendingBetsDetails()`

**ข้อมูลที่ดึงมา:**
```javascript
{
  timestamp: "2024-03-04T10:30:00.000Z",
  slipName: "ลูกชายภูน้อย",
  side: "ชล",
  amount: 150,
  opponent: "Jane",
  opponentAmount: 100,
  status: "PENDING"
}
```

**ข้อความแจ้งเตือน:**
```
⏳ รายการเงินค้างที่ยังไม่มีผล:

1. ลูกชายภูน้อย
   ฝั่ง: ชล
   เงิน: 150 บาท
   คู่ต่อสู้: Jane
   เวลา: 4/3/2567 17:30:00

📊 รวมเงินค้าง: 150 บาท
```

## 7️⃣ สรุปการคำนวนยอดเงิน

```
ระหว่างเล่น:
  ยอดเงินที่สามารถใช้ได้ = ยอดเงินคงเหลือ - เงินค้าง
  
หลังผลออก:
  ยอดเงินใหม่ = ยอดเงินเดิม + ผลลัพธ์ (WIN/LOSE/DRAW)
  
ผลลัพธ์:
  WIN: ได้รับ = ยอดเงิน - ค่าธรรมเนียม (10%)
  LOSE: เสีย = -ยอดเงิน
  DRAW: เสีย = -ค่าธรรมเนียม (5%)
```

---

## 8️⃣ ฟังก์ชันที่เกี่ยวข้อง

| ฟังก์ชัน | ไฟล์ | หน้าที่ |
|---------|------|--------|
| `getPendingAmount()` | `pendingBalanceService.js` | คำนวนเงินค้าง |
| `checkSufficientBalance()` | `pendingBalanceService.js` | ตรวจสอบเงินเพียงพอ |
| `getPendingBetsDetails()` | `pendingBalanceService.js` | ดึงรายละเอียดเงินค้าง |
| `updateBalance()` | `balanceUpdateService.js` | อัปเดตยอดเงิน |
| `updateBalancesForResult()` | `balanceUpdateService.js` | อัปเดตยอดเงินหลังผลออก |
| `recordTransaction()` | `balanceUpdateService.js` | บันทึกธุรกรรม |
