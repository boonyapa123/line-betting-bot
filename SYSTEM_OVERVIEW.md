# 🎯 ระบบการทำงานทั้งหมด

## 📋 ภาพรวมระบบ

ระบบนี้เป็นแพลตฟอร์มการเดิมพันบั้งไฟผ่าน LINE OA ที่มีการตรวจสอบสลิปอัตโนมัติ จับคู่เล่นอัตโนมัติ และคำนวนแพ้ชนะแบบเรียลไทม์

---

## 🔄 ขั้นตอนการทำงาน

### **ขั้นตอนที่ 1: ผู้ใช้ฝากเงิน (Slip Verification)**

```
ผู้ใช้ → ส่งรูปภาพสลิป → LINE OA (Account 3)
         ↓
    Webhook /webhook
         ↓
    1. ตรวจสอบลายเซ็น (Signature Validation)
    2. ดาวน์โหลดรูปภาพจาก LINE API
    3. ส่งรูปภาพไปตรวจสอบกับ Slip2Go API
       POST https://connect.slip2go.com/api/verify-slip/qr-image/info
    4. ดึงชื่อ LINE Profile ของผู้ใช้
    5. ส่ง Auto Reply (✅ ได้รับเงิน)
    6. บันทึกลงชีท:
       - Players Sheet (ชื่อ, ยอดเงิน, Token)
       - Transactions Sheet (ประวัติฝาก)
```

**ข้อมูลที่เก็บ:**
- User ID, ชื่อ LINE, ยอดเงิน, Access Token
- Reference ID, จำนวนเงิน, วันที่โอน

---

### **ขั้นตอนที่ 2: ผู้เล่นเดิมพัน (Betting)**

```
ผู้เล่น A → ส่งข้อความเดิมพัน → LINE Group
            "บั้งไฟ 310 ✅ 100"
            ↓
ผู้เล่น B → Reply ข้อความของ A (ใช้ LINE Quote)
            "บั้งไฟ 310 ❌ 100"
            ↓
    Webhook /webhook
            ↓
    1. ตรวจสอบลายเซ็น
    2. ตรวจสอบว่าเป็น Pair (User A reply User B)
    3. ดึงชื่อ LINE Profile ทั้งสองคน
    4. ดึงยอดเงินของผู้เล่นทั้งสอง
    5. ตรวจสอบยอดเงิน:
       - ถ้าเพียงพอ → บันทึกลงชีท Bets
       - ถ้าไม่พอ → ส่งข้อความแจ้งยอดเงินไม่เพียงพอ
    6. บันทึกลงชีท Bets (ข้อมูลการเดิมพัน)
```

**ข้อมูลที่เก็บ:**
- User A/B ID, ชื่อ, ข้อความ, ประเภทเดิมพัน, จำนวนเงิน
- ชื่อบั้งไฟ, กลุ่มแชท

---

### **ขั้นตอนที่ 3: ประกาศผลบั้งไฟ (Result Announcement)**

```
ผู้ประกาศ → ส่งข้อความผล → LINE Group
            "บั้งไฟ 310 ✅"
            ↓
    Webhook /webhook
            ↓
    1. ตรวจสอบลายเซ็น
    2. ค้นหาผู้เล่นที่เล่นบั้งไฟ 310
    3. ดึงยอดเงินของผู้เล่นทั้งหมด
    4. จับคู่เล่นอัตโนมัติ:
       - ค้นหาผู้เล่นที่เล่นบั้งไฟชื่อเดียวกัน
       - ยึดจากคนยอดน้อยกว่า
       - ตรวจสอบยอดเงินเพียงพอหรือไม่
    5. สำหรับแต่ละคู่ที่จับคู่สำเร็จ:
       a) คำนวนแพ้ชนะ:
          - ชนะ: ได้เงิน - 10% commission
          - แพ้: เสีย -100%
          - เสมอ: เสีย 5% commission
       b) อัปเดตผลลัพธ์ในชีท Bets
       c) ส่งข้อความแจ้งผลให้ผู้เล่น A
       d) ส่งข้อความแจ้งผลให้ผู้เล่น B
```

**ข้อมูลที่อัปเดต:**
- ผลลัพธ์ (✅/❌/⛔️) ในชีท Bets
- ส่งข้อความแจ้งผลให้ผู้เล่นทั้งสองฝั่ง

  - ✅ ตรวจสอบยอดเงินของผู้เล่น
