# กติกาการเล่นและขั้นตอน - Game Rules & Steps

## 📋 ภาพรวมกติกา

ระบบการเล่นพนันบั้งไฟ LINE Bot มีกติกาดังนี้:

---

## 🎮 ขั้นตอนการเล่น (Step by Step)

### Phase 1: ผู้เล่นเตรียมตัว

#### Step 1: ส่งสลิปเงินฝาก
```
ผู้เล่น → ส่งสลิปการโอนเงิน → LINE OA Account 3
```

**ข้อมูลที่ต้อง:**
- สลิปการโอนเงิน (QR Code)
- จำนวนเงิน
- ชื่อบัญชีรับ

**ระบบจะ:**
1. ตรวจสอบสลิป (Slip2Go)
2. ตรวจสอบจำนวนเงิน
3. ตรวจสอบชื่อบัญชีรับ
4. บันทึกลงชีท Players
5. บันทึกลงชีท Transactions
6. อัปเดตชีท UsersBalance
7. ส่งข้อความยืนยัน

**ตัวอย่าง:**
```
ผู้เล่น: [ส่งสลิป 1000 บาท]
System: ✅ ยืนยันการฝากเงินสำเร็จ
        ยอดเงินปัจจุบัน: 1000 บาท
```

---

### Phase 2: Admin เปิดรอบการเล่น

#### Step 2: Admin ส่งคำสั่งเปิดรอบ
```
Admin → :เริ่ม บั้งไฟ1 → LINE OA Account 1 หรือ Account 2
```

**คำสั่ง:**
```
:เริ่ม บั้งไฟ1
```

**ระบบจะ:**
1. ตรวจสอบรูปแบบคำสั่ง
2. สร้าง Round ID
3. บันทึกลงชีท RoundState
4. เปลี่ยนสถานะเป็น OPEN
5. ส่งข้อความยืนยัน

**ตัวอย่าง:**
```
Admin: :เริ่ม บั้งไฟ1
System: ✅ เปิดรอบการเล่น: บั้งไฟ1
        รอบ ID: ROUND_1705756800000
```

---

### Phase 3: ผู้เล่นเล่น

#### Step 3: ผู้เล่นส่งข้อความเล่น
```
ผู้เล่น → ส่งข้อความเล่น → LINE OA Account 1 หรือ Account 2
```

**รูปแบบการเล่น:**

##### วิธีที่ 1: Direct Method (ไม่มีราคา)
```
ต 500
ล 500
ชล 500
ชถ 500
ย 500
```

**ความหมาย:**
- `ต` = ไล่ (ต่ำ)
- `ล` = ไล่ (สูง)
- `ชล` = ชล (ต่ำ)
- `ชถ` = ชถ (สูง)
- `ย` = ยั้ง (กลาง)
- `500` = จำนวนเงิน

##### วิธีที่ 2: Direct Method (มีราคา)
```
40-50 ล 500
50-60 ต 500
```

**ความหมาย:**
- `40-50` = ช่วงราคา
- `ล` = ไล่ (ถ้าคะแนนอยู่ในช่วง)
- `ต` = ไล่ (ถ้าคะแนนอยู่นอกช่วง)
- `500` = จำนวนเงิน

##### วิธีที่ 3: Reply Method
```
ตอบ
```

**ความหมาย:**
- `ตอบ` = เล่นตามที่ Admin ส่ง

**ระบบจะ:**
1. Parse ข้อความ
2. ตรวจสอบรูปแบบ
3. ตรวจสอบ RoundState (OPEN?)
4. ตรวจสอบยอดเงิน (เพียงพอ?)
5. บันทึกลงชีท Bets
6. ส่งข้อความยืนยัน

**ตัวอย่าง:**
```
ผู้เล่น: ต 500
System: ✅ บันทึกการเล่นสำเร็จ
        ชื่อ: สมชาย
        บั้งไฟ: บั้งไฟ1
        ฝั่ง: ต
        จำนวนเงิน: 500 บาท
```

---

### Phase 4: Admin ปิดรอบ

#### Step 4: Admin ส่งคำสั่งปิดรอบ
```
Admin → :หยุด → LINE OA Account 1 หรือ Account 2
```

**คำสั่ง:**
```
:หยุด
```

**ระบบจะ:**
1. ตรวจสอบรูปแบบคำสั่ง
2. เปลี่ยนสถานะเป็น CLOSED
3. บันทึกลงชีท RoundState
4. ส่งข้อความยืนยัน

**ตัวอย่าง:**
```
Admin: :หยุด
System: ✅ รอบนี้ปิดการทายแล้วคะ/ครับ
```

**ผลกระทบ:**
- ผู้เล่นไม่สามารถส่งข้อความเล่นได้อีก
- ระบบจะตอบ "รอบนี้ปิดการทายแล้ว"

---

### Phase 5: Admin สรุปผล

#### Step 5: Admin ส่งคำสั่งสรุปผล
```
Admin → :สรุป บั้งไฟ1 50 → LINE OA Account 1 หรือ Account 2
```

**คำสั่ง:**
```
:สรุป บั้งไฟ1 50
```

**ระบบจะ:**
1. ตรวจสอบรูปแบบคำสั่ง
2. ดึงข้อมูลการเล่นทั้งหมด
3. จับคู่การเล่น (Normal + Self-Betting)
4. คำนวณผลลัพธ์
5. คำนวณค่าธรรมเนียม
6. บันทึกลงชีท Results
7. อัปเดตยอดเงิน
8. ส่งข้อความแจ้งเตือน LINE
9. ล้างข้อมูลการเล่น
10. ปิดรอบ

**ตัวอย่าง:**
```
Admin: :สรุป บั้งไฟ1 50
System: 📊 สรุปผลการเล่น
        บั้งไฟ: บั้งไฟ1
        คะแนนที่ออก: 50
        ========================================
        
        คู่ที่ 1:
        🏆 ชนะ: สมชาย +450 บาท
        ❌ แพ้: สมหญิง -500 บาท
        
        ========================================
        💰 ยอดเงินคงเหลือ:
        
        สมชาย: 1450 บาท
        สมหญิง: 500 บาท
```

## 💰 กติกาการคำนวณเงิน

### 1. การเล่นปกติ (Normal Betting)

#### ชนะ-แพ้ (Win-Lose)
```
ผู้เล่น A เดิมพัน: 500 บาท (ฝั่ง ต)
ผู้เล่น B เดิมพัน: 500 บาท (ฝั่ง ล)

คะแนนออก: 50 (ฝั่ง ต ชนะ)

ผู้ชนะ (A):
- เดิมพัน: 500 บาท
- ค่าธรรมเนียม: 50 บาท (10%)
- ได้รับ: 450 บาท

ผู้แพ้ (B):
- เดิมพัน: -500 บาท
- ค่าธรรมเนียม: 0 บาท
- เสีย: -500 บาท

ยอดเงินหลังเล่น:
- A: 1000 + 450 = 1450 บาท
- B: 1000 - 500 = 500 บาท
```

#### ออกกลาง (Draw)
```
ผู้เล่น A เดิมพัน: 500 บาท (ฝั่ง ต)
ผู้เล่น B เดิมพัน: 500 บาท (ฝั่ง ล)

คะแนนออก: 50 (ทั้งสองฝั่งชนะ = ออกกลาง)

ทั้งสองฝั่ง:
- เดิมพัน: 500 บาท
- ค่าธรรมเนียม: 25 บาท (5%)
- เสีย: -25 บาท

ยอดเงินหลังเล่น:
- A: 1000 - 25 = 975 บาท
- B: 1000 - 25 = 975 บาท
```

### 2. การเล่นกับตัวเอง (Self-Betting)

```
ผู้เล่น A เดิมพัน: 500 บาท (ฝั่ง ต)
ผู้เล่น A เดิมพัน: 500 บาท (ฝั่ง ล)

คะแนนออก: 50 (ฝั่ง ต ชนะ)

ผู้เล่น A:
- ฝั่ง ต ชนะ: +450 บาท
- ฝั่ง ล แพ้: -500 บาท
- รวม: -50 บาท (หักค่าธรรมเนียม)

ยอดเงินหลังเล่น:
- A: 1000 - 50 = 950 บาท
```

## ✅ เงื่อนไขการเล่น

### 1. ยอดเงินต้องเพียงพอ
```
ยอดเงินปัจจุบัน >= จำนวนเงินที่ต้องการเดิมพัน
```

**ตัวอย่าง:**
```
✅ ยอดเงิน 1000 บาท เล่น 500 บาท → ได้
❌ ยอดเงิน 300 บาท เล่น 500 บาท → ไม่ได้
```

### 2. รอบต้องเปิด (OPEN)
```
RoundState = OPEN
```

**ตัวอย่าง:**
```
✅ Admin เปิดรอบแล้ว → ได้เล่น
❌ Admin ยังไม่เปิดรอบ → ไม่ได้เล่น
❌ Admin ปิดรอบแล้ว → ไม่ได้เล่น
```

### 3. รูปแบบข้อความต้องถูกต้อง
```
ต 500 ✅
ล 500 ✅
40-50 ล 500 ✅
ตอบ ✅

ต500 ❌ (ไม่มีช่องว่าง)
500ต ❌ (ลำดับผิด)
ต ❌ (ไม่มีจำนวนเงิน)
```

## 🎯 สรุปกติกา

1. **ฝากเงิน** → ส่งสลิป → ระบบตรวจสอบ → บันทึกยอดเงิน
2. **เปิดรอบ** → Admin `:เริ่ม` → ระบบเปิด
3. **เล่น** → ผู้เล่นส่งข้อความ → ระบบตรวจสอบ → บันทึก
4. **ปิดรอบ** → Admin `:หยุด` → ระบบปิด
5. **สรุปผล** → Admin `:สรุป` → ระบบคำนวณ → แจ้งเตือน

---

## ⚠️ ข้อจำกัด

- ยอดเงินต้องเพียงพอ
- รอบต้องเปิด
- รูปแบบข้อความต้องถูกต้อง
- ค่าธรรมเนียมหัก 10% (ชนะ-แพ้) หรือ 5% (ออกกลาง)
