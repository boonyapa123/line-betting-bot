# 🎮 ภาพรวมระบบการเล่นทั้งหมด

## 🎯 ระบบคืออะไร?

ระบบตรวจสอบสลิปจาก LINE OA เป็นระบบอัตโนมัติที่ช่วยให้:
- ผู้ใช้ส่งรูปภาพสลิปธนาคารไปยัง LINE OA
- ระบบตรวจสอบความถูกต้องของสลิปโดยอัตโนมัติ
- บันทึกข้อมูลสลิปลง Google Sheets
- ส่งข้อความตอบกลับไปยังผู้ใช้

## 🎮 ผู้เล่นในระบบ

### 1. 👤 ผู้ใช้ (User)
**บทบาท:** ส่งรูปภาพสลิป

**ขั้นตอน:**
1. เปิด LINE OA
2. ส่งรูปภาพสลิปธนาคาร
3. รอรับข้อความตอบกลับ
4. ตรวจสอบข้อมูลสลิป

**ตัวอย่าง:**
```
ผู้ใช้: [ส่งรูปภาพสลิป]
LINE OA: ✅ ได้รับยอดเงินแล้ว
         💰 จำนวนเงิน: 1000 บาท
         👤 ผู้ส่ง: สมชาย สลิปทูโก
         👥 ผู้รับ: บริษัท สลิปทูโก จำกัด
```

---

### 2. 📱 LINE OA (Official Account)
**บทบาท:** ตัวกลางระหว่างผู้ใช้และ Server

**ขั้นตอน:**
1. รับรูปภาพจากผู้ใช้
2. บันทึก Event (Webhook)
3. ส่ง Webhook ไปยัง Server
4. รับข้อความตอบกลับจาก Server
5. ส่งข้อความตอบกลับไปยังผู้ใช้

**ตัวอย่าง:**
```
ผู้ใช้ → LINE OA → [บันทึก Event]
                    ↓
                    ส่ง Webhook
                    ↓
                    Server
                    ↓
                    ได้ข้อความตอบกลับ
                    ↓
                    ส่งไปยังผู้ใช้
```

---

### 3. 🖥️ Server (Node.js Application)
**บทบาท:** ประมวลผลข้อมูลและตรวจสอบสลิป

**ขั้นตอน:**
1. รับ Webhook จาก LINE OA
2. ดาวน์โหลดรูปภาพ
3. สแกน QR Code
4. ตรวจสอบสลิปผ่าน Slip2Go API
5. ตรวจสอบสลิปซ้ำใน Google Sheets
6. บันทึกข้อมูลลง Google Sheets
7. สร้างข้อความตอบกลับ
8. ส่งข้อความตอบกลับไปยัง LINE OA

**ตัวอย่าง:**
```
Server:
  1. รับ Webhook ✅
  2. ดาวน์โหลดรูปภาพ ✅
  3. สแกน QR Code ✅
  4. ตรวจสอบสลิป ✅
  5. ตรวจสอบซ้ำ ✅
  6. บันทึก ✅
  7. สร้างข้อความ ✅
  8. ส่งข้อความ ✅
```

---

### 4. 🔐 Slip2Go API
**บทบาท:** ตรวจสอบความถูกต้องของสลิป

**ขั้นตอน:**
1. รับ QR Code String จาก Server
2. ตรวจสอบในระบบธนาคาร
3. ส่งข้อมูลสลิปกลับมา

**ตัวอย่าง:**
```
Server → Slip2Go API
  ↓
  QR Code: "0041000600000101030040220014242082547BPM049885102TH9104xxxx"
  ↓
  Slip2Go ตรวจสอบ
  ↓
  ส่งกลับ:
  {
    "code": "200000",
    "amount": 1000,
    "sender": "สมชาย สลิปทูโก",
    "receiver": "บริษัท สลิปทูโก จำกัด"
  }
```

---

### 5. 📊 Google Sheets
**บทบาท:** บันทึกข้อมูลสลิป

**ขั้นตอน:**
1. รับข้อมูลสลิปจาก Server
2. ตรวจสอบว่า Worksheet มีอยู่
3. สร้าง Worksheet ถ้าไม่มี
4. บันทึกข้อมูลแถวใหม่

**ตัวอย่าง:**
```
Google Sheets:
┌─────────────┬──────────────┬──────────────┬──────────┐
│ วันที่บันทึก │ Reference ID │ จำนวนเงิน    │ ชื่อผู้ส่ง │
├─────────────┼──────────────┼──────────────┼──────────┤
│ 26/02/2025  │ 92887bd5-... │ 1000         │ สมชาย    │
│ 26/02/2025  │ 92887bd6-... │ 2000         │ สมชาย    │
│ 26/02/2025  │ 92887bd7-... │ 500          │ สมชาย    │
└─────────────┴──────────────┴──────────────┴──────────┘
```

## 🔄 ขั้นตอนการทำงานทั้งหมด

### Phase 1: ผู้ใช้ส่งรูปภาพ (0-1 วินาที)

```
ผู้ใช้ → LINE OA
  ↓
  ส่งรูปภาพสลิป
  ↓
  LINE OA บันทึก Event
```

**สิ่งที่เกิดขึ้น:**
- ผู้ใช้เปิด LINE OA
- ส่งรูปภาพสลิปธนาคาร
- LINE OA รับรูปภาพ

---

### Phase 2: LINE OA ส่ง Webhook (1-2 วินาที)

```
LINE OA → Server
  ↓
  POST /webhook/line-slip-verification
  ↓
  Body: {
    "events": [{
      "type": "message",
      "message": {
        "type": "image",
        "id": "100001"
      }
    }]
  }
```

**สิ่งที่เกิดขึ้น:**
- LINE OA ส่ง Webhook ไปยัง Server
- Server รับ Webhook
- ตรวจสอบว่าเป็น Image Message

---

### Phase 3: ดาวน์โหลดรูปภาพ (2-3 วินาที)

```
Server → LINE API
  ↓
  GET /v2/bot/message/{messageId}/content
  ↓
  ได้ Image Buffer
```

**สิ่งที่เกิดขึ้น:**
- Server ใช้ messageId เพื่อดึงรูปภาพ
- ได้ Buffer ของรูปภาพ
- เก็บไว้ในหน่วยความจำ

---

### Phase 4: สแกน QR Code (3-5 วินาที)

```
QRCodeScannerService
  ↓
  1. อ่านรูปภาพ (Jimp)
  2. แปลงเป็น ImageData
  3. สแกน QR Code (jsQR)
  ↓
  ได้ QR Code String
```

**สิ่งที่เกิดขึ้น:**
- ประมวลผลรูปภาพ
- สแกน QR Code
- ได้ QR Code String

**ตัวอย่าง:**
```
QR Code: "0041000600000101030040220014242082547BPM049885102TH9104xxxx"
```

---

### Phase 5: ตรวจสอบสลิป (5-7 วินาที)

```
Server → Slip2Go API
  ↓
  POST /api/verify-slip/qr-code/info
  ↓
  Body: {
    "payload": {
      "qrCode": "0041000600000101030040220014242082547BPM049885102TH9104xxxx",
      "checkCondition": {
        "checkDuplicate": true,
        "checkReceiver": [...]
      }
    }
  }
  ↓
  Response: {
    "code": "200000",
    "data": {
      "amount": 1000,
      "sender": {...},
      "receiver": {...}
    }
  }
```

**สิ่งที่เกิดขึ้น:**
- ส่ง QR Code ไปยัง Slip2Go API
- Slip2Go ตรวจสอบในระบบธนาคาร
- ได้ผลลัพธ์

---

### Phase 6: ตรวจสอบสลิปซ้ำ (7-8 วินาที)

```
SlipRecordingService
  ↓
  ค้นหา referenceId ใน Google Sheets
  ↓
  ถ้าพบ → สลิปซ้ำ ❌
  ถ้าไม่พบ → สลิปใหม่ ✅
```

**สิ่งที่เกิดขึ้น:**
- ค้นหา referenceId ใน Google Sheets
- ตรวจสอบว่าสลิปซ้ำหรือไม่

---

### Phase 7: บันทึกข้อมูล (8-10 วินาที)

```
SlipRecordingService
  ↓
  1. ตรวจสอบ Worksheet
  2. สร้าง Worksheet (ถ้าไม่มี)
  3. เพิ่ม Header (ถ้าเป็นครั้งแรก)
  4. บันทึกข้อมูลแถวใหม่
  ↓
  Google Sheets API
  ↓
  Append values to sheet
```

**สิ่งที่เกิดขึ้น:**
- บันทึกข้อมูลลง Google Sheets
- สร้าง Worksheet ถ้าไม่มี
- เพิ่ม Header ถ้าเป็นครั้งแรก

---

### Phase 8: สร้างข้อความตอบกลับ (10-11 วินาที)

```
LineSlipVerificationService
  ↓
  ตรวจสอบผลลัพธ์
  ↓
  สร้างข้อความ:
  ✅ ได้รับยอดเงินแล้ว
  📊 รายละเอียดสลิป
  💰 จำนวนเงิน: 1000 บาท
  👤 ผู้ส่ง: สมชาย สลิปทูโก
  👥 ผู้รับ: บริษัท สลิปทูโก จำกัด
  📅 วันที่: 26/02/2025 14:30:45
```

**สิ่งที่เกิดขึ้น:**
- ตรวจสอบผลลัพธ์
- สร้างข้อความตอบกลับ

---

### Phase 9: ส่งข้อความตอบกลับ (11-12 วินาที)

```
Server → LINE API
  ↓
  POST /v1/bot/message/push
  ↓
  Body: {
    "to": "<USER_ID>",
    "messages": [{
      "type": "text",
      "text": "✅ ได้รับยอดเงินแล้ว\n..."
    }]
  }
  ↓
  ผู้ใช้ได้รับข้อความ
```

**สิ่งที่เกิดขึ้น:**
- ส่งข้อความตอบกลับไปยัง LINE OA
- ผู้ใช้ได้รับข้อความ

## 🔐 ความปลอดภัย

### 1. Authentication
- **LINE:** ตรวจสอบ Channel Secret
- **Slip2Go:** ใช้ Bearer Token
- **Google Sheets:** ใช้ Service Account

### 2. Data Protection
- Secret Key เก็บใน `.env`
- ไม่ commit credentials ไปยัง Git
- ใช้ HTTPS สำหรับ Webhook URL

### 3. Validation
- ตรวจสอบ Event Type
- ตรวจสอบ Message Type
- ตรวจสอบ QR Code Format
