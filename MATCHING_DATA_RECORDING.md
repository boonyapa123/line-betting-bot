# 📝 การบันทึกข้อมูลการจับคู่ลงชีท Bets

## 🔄 ขั้นตอนการบันทึก

### Step 1: User A ส่งข้อความเดิมพัน

```
User A: "350-370 ล 300 ฟ้า"
```

**ฟังก์ชัน**: `recordBet()` ใน `bettingPairingService.js`

**บันทึกคอลัมน์:**

| คอลัมน์ | ชื่อ | ค่า | ตัวอย่าง |
|--------|------|-----|---------|
| A | Timestamp | เวลาปัจจุบัน | 05/03/2569 03:33:26 |
| B | User A ID | ID ของ User A | Uf00f7dcba844fbc87a181897bcb863e3 |
| C | ชื่อ User A | ชื่อ LINE ของ User A | นุช519 |
| D | ข้อความ A | ข้อความเดิมพันเต็ม | 350-370 ล 300 ฟ้า |
| E | ชื่อบั้งไฟ | ชื่อบั้งไฟ | ฟ้า |
| F | รายการเล่น (ฝั่ง A) | ฝั่งของ User A | ล |
| G | ยอดเงิน | ยอดเงินเดิมพัน | 300 |
| O | ชื่อกลุ่ม | ชื่อกลุ่ม (ถ้ามี) | (ว่างเปล่า) |
| P | Token A | Token ของ User A | (ว่างเปล่า) |
| Q | ID กลุ่ม | ID ของกลุ่ม LINE | C4e522277480703e5eddbf658666ba6a9 |

**ผลลัพธ์:**
```
Row 2:
  A: 05/03/2569 03:33:26
  B: Uf00f7dcba844fbc87a181897bcb863e3
  C: นุช519
  D: 350-370 ล 300 ฟ้า
  E: ฟ้า
  F: ล
  G: 300
  H-U: (ว่างเปล่า)
```

---

### Step 2: ค้นหาคู่ (Matching)

ระบบค้นหาคู่ที่มีฝั่งตรงข้าม

**ตัวอย่าง:**
```
User B: "350-370 ย 100 ฟ้า"
```

---

### Step 3: จับคู่สำเร็จ (Update Row)

**ฟังก์ชัน**: `updateRowWithUserB()` ใน `bettingPairingService.js`

**อัปเดตคอลัมน์:**

| คอลัมน์ | ชื่อ | ค่า | ตัวอย่าง |
|--------|------|-----|---------|
| H | ยอดเงิน B | ยอดเงินเดิมพันของ User B | 100 |
| K | User B ID | ID ของ User B | Uc2a009fe53d51946657363bdbb7d1374 |
| L | ชื่อ User B | ชื่อ LINE ของ User B | 💓Noon💓 |
| M | รายการแทง (ฝั่ง B) | ฝั่งของ User B | ย |
| R | Token B | Token ของ User B | (ว่างเปล่า) |
| U | MATCHED Auto | ทำเครื่องหมายจับคู่อัตโนมัติ | AUTO |

**ผลลัพธ์:**
```
Row 2 (หลังจับคู่):
  A: 05/03/2569 03:33:26
  B: Uf00f7dcba844fbc87a181897bcb863e3
  C: นุช519
  D: 350-370 ล 300 ฟ้า
  E: ฟ้า
  F: ล
  G: 300
  H: 100 ← อัปเดต
  I: (ว่างเปล่า)
  J: (ว่างเปล่า)
  K: Uc2a009fe53d51946657363bdbb7d1374 ← อัปเดต
  L: 💓Noon💓 ← อัปเดต
  M: ย ← อัปเดต
  N-Q: (ว่างเปล่า)
  R: (ว่างเปล่า) ← อัปเดต (ยังว่างเปล่า)
  S-T: (ว่างเปล่า)
  U: AUTO ← อัปเดต
```

## 🔍 ตรวจสอบการบันทึก

### ตรวจสอบ: ยังไม่จับคู่

```
H: (ว่างเปล่า)
K-M: (ว่างเปล่า)
U: (ว่างเปล่า)
```

### ตรวจสอบ: จับคู่สำเร็จ

```
H: มีค่า (ยอดเงิน User B)
K: มีค่า (ID User B)
L: มีค่า (ชื่อ User B)
M: มีค่า (ฝั่ง User B)
U: AUTO
```

## 📌 สรุปการบันทึก

### User A บันทึก (recordBet)

**คอลัมน์ที่บันทึก:**
- A: Timestamp
- B: User A ID
- C: ชื่อ User A
- D: ข้อความ A
- E: ชื่อบั้งไฟ
- F: รายการเล่น (ฝั่ง A)
- G: ยอดเงิน
- O: ชื่อกลุ่ม
- P: Token A
- Q: ID กลุ่ม

**คอลัมน์ที่ว่างเปล่า:**
- H, I, J, K, L, M, N, R, S, T, U

---

### User B จับคู่ (updateRowWithUserB)

**คอลัมน์ที่อัปเดต:**
- H: ยอดเงิน B
- K: User B ID
- L: ชื่อ User B
- M: รายการแทง (ฝั่ง B)
- R: Token B
- U: MATCHED Auto (AUTO)

**คอลัมน์ที่ไม่เปลี่ยน:**
- A-G, I, J, N, O, P, Q, S, T

---

## 🎯 ขั้นตอนการบันทึกสมบูรณ์

```
User A ส่งข้อความ
  ↓
[recordBet] บันทึก A-G, O, P, Q
  ↓
ค้นหาคู่
  ↓
User B จับคู่
  ↓
[updateRowWithUserB] อัปเดต H, K, L, M, R, U
  ↓
✅ จับคู่สำเร็จ
  ↓
รอประกาศผล
  ↓
[recordResult] อัปเดต I, J, S, T
  ↓
✅ ประกาศผลสำเร็จ
```
