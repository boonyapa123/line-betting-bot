# โครงสร้างชีท Bets - Bets Sheet Structure

## 📊 ภาพรวม

ชีท **Bets** เป็นที่บันทึกการเล่นของผู้เล่นแต่ละครั้ง

---

## 🔍 โครงสร้างคอลัมน์

| คอลัมน์ | ชื่อ | ข้อมูล | ตัวอย่าง |
|--------|------|--------|---------|
| **A** | Timestamp | วันเวลาที่บันทึก | 2024-03-02T14:56:01.477Z |
| **B** | UserID | ID ผู้เล่น | U2fbb940e366149fc8dbdf92b92a81b60 |
| **C** | DisplayName | ชื่อผู้เล่น | 💓Noon💓 |
| **D** | LineName | ชื่อ LINE | 💓Noon💓 |
| **E** | Method | วิธีการเล่น | REPLY หรือ 1 หรือ 2 |
| **F** | Price | ราคา (วิธีที่ 2 เท่านั้น) | 100-200 |
| **G** | Side | ฝั่งที่เล่น | ชล/ชถ/ล/ย/ต |
| **H** | Amount | จำนวนเงิน | 150 |
| **I** | SlipName | ชื่อบั้งไฟ | ลูกชายภูน้อย |
| **J** | Status | สถานะ | OPEN/MATCHED/SETTLED |

---

## 📝 รายละเอียดแต่ละคอลัมน์

### A: Timestamp (วันเวลา)
**ทำหน้าที่:** บันทึกเวลาที่ผู้เล่นเล่น

**ข้อมูล:** ISO 8601 format
```
2024-03-02T14:56:01.477Z
```

**ตัวอย่าง:**
```
2024-03-02T14:56:01.477Z
2024-03-02T14:56:36.041Z
```

---

### B: UserID (ID ผู้เล่น)
**ทำหน้าที่:** ระบุตัวตนของผู้เล่น

**ข้อมูล:** LINE User ID
```
U2fbb940e366149fc8dbdf92b92a81b60
```

**ตัวอย่าง:**
```
U2fbb940e366149fc8dbdf92b92a81b60
Ubb4a196cd0dc0c3ca6211a9bafecb513
```

---

### C: DisplayName (ชื่อผู้เล่น)
**ทำหน้าที่:** แสดงชื่อผู้เล่น

**ข้อมูล:** ชื่อที่ LINE แสดง
```
💓Noon💓
```

**ตัวอย่าง:**
```
💓Noon💓
Pam Yuthida
นุช519
```

---

### D: LineName (ชื่อ LINE)
**ทำหน้าที่:** ชื่อ LINE สำหรับตรวจสอบยอดเงิน

**ข้อมูล:** ชื่อ LINE
```
💓Noon💓
```

**ตัวอย่าง:**
```
💓Noon💓
Pam Yuthida
นุช519
```

---

### E: Method (วิธีการเล่น)
**ทำหน้าที่:** ระบุวิธีการเล่น

**ข้อมูล:** 
- `REPLY` - ตอบกลับข้อความ
- `1` - ลงเล่นตรง (วิธีที่ 1)
- `2` - ลงเล่นตรง (วิธีที่ 2)

**ตัวอย่าง:**
```
REPLY
1
2
```

**ความหมาย:**
- **REPLY** - ผู้เล่นตอบกลับข้อความของผู้เล่นอื่น
- **1** - ผู้เล่นลงเล่นตรง (Direct Method)
- **2** - ผู้เล่นลงเล่นตรง (Direct Method)

---

### F: Price (ราคา)
**ทำหน้าที่:** บันทึกราคา (วิธีที่ 2 เท่านั้น)

**ข้อมูล:** ช่วงราคา
```
100-200
50-100
```

**ตัวอย่าง:**
```
100-200
50-100
(ว่างเปล่า - ถ้าไม่ใช่วิธีที่ 2)
```

**หมายเหตุ:** 
- วิธีที่ 1 (REPLY) - ไม่มีราคา
- วิธีที่ 2 (Direct) - อาจมีราคา

---

### G: Side (ฝั่งที่เล่น)
**ทำหน้าที่:** ระบุฝั่งที่เล่น

**ข้อมูล:** 
- `ชล` - ไล่
- `ชถ` - ถอย
- `ล` - ไล่
- `ย` - ยาว
- `ต` - ตรงข้าม (Reply Method)

**ตัวอย่าง:**
```
ชล
ชถ
ล
ย
ต
```

**ความหมาย:**
- **ชล** - ไล่ (ตัวเลขสูง)
- **ชถ** - ถอย (ตัวเลขต่ำ)
- **ล** - ไล่
- **ย** - ยาว
- **ต** - ตรงข้าม (ใช้เมื่อ Reply)

---

### H: Amount (จำนวนเงิน)
**ทำหน้าที่:** บันทึกจำนวนเงินที่เล่น

**ข้อมูล:** ตัวเลข (บาท)
```
150
500
1000
```

**ตัวอย่าง:**
```
150
500
1000
(ว่างเปล่า - ถ้าเป็น REPLY Method)
```

**หมายเหตุ:**
- Direct Method - มีจำนวนเงิน
- Reply Method - ไม่มีจำนวนเงิน (รอการจับคู่)

---

### I: SlipName (ชื่อบั้งไฟ)
**ทำหน้าที่:** ระบุบั้งไฟที่เล่น

**ข้อมูล:** ชื่อบั้งไฟ
```
ลูกชายภูน้อย
บั้งไฟ1
ฟ้าหลังฝน
```

**ตัวอย่าง:**
```
ลูกชายภูน้อย
บั้งไฟ1
ฟ้าหลังฝน
```

**หมายเหตุ:**
- ต้องตรงกับชื่อบั้งไฟที่ Admin ประกาศ
- ใช้สำหรับจับคู่ผู้เล่น

---

### J: Status (สถานะ)
**ทำหน้าที่:** ระบุสถานะการเล่น

**ข้อมูล:**
- `OPEN` - รอการจับคู่
- `MATCHED` - จับคู่สำเร็จ
- `SETTLED` - สรุปผลแล้ว

**ตัวอย่าง:**
```
OPEN
MATCHED
SETTLED
```

**ความหมาย:**
- **OPEN** - บันทึกแล้ว รอการจับคู่
- **MATCHED** - จับคู่กับผู้เล่นอื่นแล้ว
- **SETTLED** - สรุปผลแล้ว

---

## 📋 ตัวอย่างข้อมูลในชีท Bets

### ตัวอย่าง 1: Direct Method (วิธีที่ 1)

```
| Timestamp | UserID | DisplayName | LineName | Method | Price | Side | Amount | SlipName | Status |
|-----------|--------|-------------|----------|--------|-------|------|--------|----------|--------|
| 2024-03-02T14:56:01.477Z | U2fbb940e366149fc8dbdf92b92a81b60 | 💓Noon💓 | 💓Noon💓 | 1 | | ชล | 150 | ลูกชายภูน้อย | OPEN |
```

**ความหมาย:**
- 💓Noon💓 ลงเล่นตรง
- ฝั่ง: ชล (ไล่)
- จำนวนเงิน: 150 บาท
- บั้งไฟ: ลูกชายภูน้อย
- สถานะ: รอการจับคู่

---

### ตัวอย่าง 2: Reply Method

```
| Timestamp | UserID | DisplayName | LineName | Method | Price | Side | Amount | SlipName | Status |
|-----------|--------|-------------|----------|--------|-------|------|--------|----------|--------|
| 2024-03-02T14:56:01.477Z | U2fbb940e366149fc8dbdf92b92a81b60 | 💓Noon💓 | 💓Noon💓 | 1 | | ชล | 150 | ลูกชายภูน้อย | OPEN |
| 2024-03-02T14:56:36.041Z | Ubb4a196cd0dc0c3ca6211a9bafecb513 | Pam Yuthida | Pam Yuthida | REPLY | | ต | | ลูกชายภูน้อย | MATCHED |
```

**ความหมาย:**
- Row 1: 💓Noon💓 ลงเล่นตรง ชล 150 บาท
- Row 2: Pam Yuthida ตอบกลับ (ฝั่งตรงข้าม = ชถ)
- สถานะ: MATCHED (จับคู่สำเร็จ)

---

### ตัวอย่าง 3: Direct + Direct Method (วิธีที่ 2)

```

```

**ความหมาย:**
- Row 1: 💓Noon💓 ลงเล่นตรง ชล 500 บาท
- Row 2: Pam Yuthida ลงเล่นตรง ชถ 300 บาท
- สถานะ: MATCHED (จับคู่สำเร็จ)
- ยอดเดิมพัน: 300 บาท (ยอดน้อยกว่า)

## ✅ ตรวจสอบข้อมูล

### ตรวจสอบ 1: ข้อมูลครบถ้วน
```
✅ Timestamp - มี
✅ UserID - มี
✅ DisplayName - มี
✅ LineName - มี
✅ Method - มี
✅ Side - มี
✅ Amount - มี (Direct) หรือ ว่าง (Reply)
✅ SlipName - มี
✅ Status - มี
```

### ตรวจสอบ 2: ข้อมูลถูกต้อง
```
✅ Timestamp - ISO 8601 format
✅ UserID - LINE User ID
✅ Method - REPLY, 1, หรือ 2
✅ Side - ชล, ชถ, ล, ย, หรือ ต
✅ Amount - ตัวเลข (Direct) หรือ ว่าง (Reply)
✅ Status - OPEN, MATCHED, หรือ SETTLED
```

### ตรวจสอบ 3: ข้อมูลสอดคล้อง
```
✅ SlipName ตรงกับ RoundState
✅ Side ตรงข้ามกัน (ชล ↔ ชถ)
✅ Amount เท่ากัน (Direct + Direct)
✅ Status อัปเดตตามขั้นตอน
```
