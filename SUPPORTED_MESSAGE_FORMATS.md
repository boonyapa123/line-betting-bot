# รูปแบบข้อความที่ระบบรองรับ

## 📋 สรุปรูปแบบข้อความทั้งหมด

ระบบรองรับข้อความเล่นทั้ง 2 วิธี:
- **วิธีที่ 1**: ราคาช่าง (ชล./ชถ.)
- **วิธีที่ 2**: ราคาคะแนน (ล./ย.)

---

## 🎯 วิธีที่ 1: ราคาช่าง (ชล./ชถ.)

### รูปแบบ 1.1: ชื่อบั้งไฟ + ชล./ชถ. + ยอดเงิน
```
ฟ้าหลังฝน ชล. 500
```
- **ชื่อบั้งไฟ**: ฟ้าหลังฝน
- **ฝั่ง**: ชล. (ชล = ไล่)
- **ยอดเงิน**: 500 บาท

### รูปแบบ 1.2: ชล/ชถ/ชย + ยอดเงิน + ชื่อบั้งไฟ
```
ชล 500 ฟ้าหลังฝน
ชถ 300 คำไผ่
ชย 200 เป็ด
```
- **ฝั่ง**: ชล (ไล่), ชถ (ยั้ง), ชย (ยั้ง)
- **ยอดเงิน**: 500, 300, 200 บาท
- **ชื่อบั้งไฟ**: ฟ้าหลังฝน, คำไผ่, เป็ด

---

## 🎯 วิธีที่ 2: ราคาคะแนน (ล./ย.)

### รูปแบบ 2.1: ราคา + ล./ย. + ยอดเงิน + ชื่อบั้งไฟ
```
300-330 ล 500 ฟ้าหลังฝน
350-370 ย. 300 คำไผ่
```
- **ราคา**: 300-330, 350-370
- **ฝั่ง**: ล (ไล่), ย (ยั้ง)
- **ยอดเงิน**: 500, 300 บาท
- **ชื่อบั้งไฟ**: ฟ้าหลังฝน, คำไผ่

### รูปแบบ 2.2: ราคา + ล/ย + ยอดเงิน + ชื่อบั้งไฟ (ไม่มีจุด)
```
300-330 ล 500 ฟ้าหลังฝน
350-370 ย 300 คำไผ่
```
- **ราคา**: 300-330, 350-370
- **ฝั่ง**: ล (ไล่), ย (ยั้ง)
- **ยอดเงิน**: 500, 300 บาท
- **ชื่อบั้งไฟ**: ฟ้าหลังฝน, คำไผ่

### รูปแบบ 2.3: Slash Format - [ฝั่ง]/[ราคา]/[ยอดเงิน][ชื่อบั้งไฟ]
```
ไล่/360-400/20เป็ด
ต/370-410/50เป็ด
ล/325-340/100ฟ้า
ย/300-330/200คำไผ่
```
- **ฝั่ง**: ไล่ (ล), ต (ย), ล, ย
- **ราคา**: 360-400, 370-410, 325-340, 300-330
- **ยอดเงิน**: 20, 50, 100, 200 บาท
- **ชื่อบั้งไฟ**: เป็ด, เป็ด, ฟ้า, คำไผ่

### รูปแบบ 2.4: Slash Format - [ฝั่ง]/[ราคา]/[ยอดเงิน]/[ชื่อบั้งไฟ] (มี / ระหว่าง amount และ slip name)
```
ไล่/360-400/50/เป็ด
ต/370-410/40/เป็ด
ล/325-340/100/ฟ้า
ย/300-330/200/คำไผ่
```
- **ฝั่ง**: ไล่ (ล), ต (ย), ล, ย
- **ราคา**: 360-400, 370-410, 325-340, 300-330
- **ยอดเงิน**: 50, 40, 100, 200 บาท
- **ชื่อบั้งไฟ**: เป็ด, เป็ด, ฟ้า, คำไผ่

---

## 🔄 Reply Method

### รูปแบบ 3.1: Simple Reply - เพียงแค่ "ต" หรือ "ต."
```
ต
ต.
```
- ระบบจะใช้ข้อมูลของ User A
- คำนวณฝั่งตรงข้าม
- ยอดเงินเท่ากับ User A

### รูปแบบ 3.2: Direct Method Reply - ส่งข้อความเล่นแบบเดียวกับ User A
```
ไล่/360-400/20เป็ด
ล/325-340/100ฟ้า
300-330 ล 500 ฟ้าหลังฝน
```
- ระบบจะ parse ข้อความ reply
- ใช้ข้อมูลจากข้อความ reply
- ใช้ slip name เดียวกับ User A

## ✅ ฝั่งที่รองรับ

### วิธีที่ 1 (ราคาช่าง)
- `ชล` / `ชล.` = ไล่ (สูง)
- `ชถ` / `ชถ.` = ยั้ง (ต่ำ)
- `ชย` / `ชย.` = ยั้ง (ต่ำ)

### วิธีที่ 2 (ราคาคะแนน)
- `ล` / `ล.` = ไล่ (สูง)
- `ย` / `ย.` = ยั้ง (ต่ำ)

### Slash Format
- `ไล่` / `ล` / `ส` = ไล่ (สูง)
- `ยั้ง` / `ต` / `ย` = ยั้ง (ต่ำ)

---

## 💡 หมายเหตุ

1. **ชื่อบั้งไฟ**: ต้องเป็นตัวอักษรไทยเท่านั้น (ไม่มีตัวเลข)
2. **ราคา**: รูปแบบ `XXX-XXX` (เช่น 300-330, 360-400)
3. **ยอดเงิน**: ต้องเป็นตัวเลขเท่านั้น
4. **ฝั่ง**: ต้องตรงกับรูปแบบที่กำหนด
5. **Reply**: ถ้าส่งข้อความเล่นแบบเดียวกับ User A ระบบจะใช้ข้อมูลจากข้อความ reply แต่ใช้ slip name เดียวกับ User A
